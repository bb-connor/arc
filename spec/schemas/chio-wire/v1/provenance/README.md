# Chio Wire Schemas v1: provenance/

This subtree contains the machine-readable JSON Schema (draft 2020-12) contract for the four provenance-graph primitives that anchor `spec/PROTOCOL.md` provenance prose and the M07 tool-call fabric: `stamp.schema.json` is the wire form of the M07 `ProvenanceStamp` (canonical fields `provider`, `request_id`, `api_version`, `principal`, `received_at` per `.planning/trajectory/01-spec-codegen-conformance.md` Cross-doc references row); `context.schema.json` mirrors the `GovernedCallChainContext` struct in `crates/chio-core-types/src/capability.rs` (lines 952-967, `serde(rename_all = camelCase)`); `attestation-bundle.schema.json` carries an ordered list of normalized runtime attestation evidence statements that mirror `RuntimeAttestationEvidence` in `crates/chio-core-types/src/capability.rs` (lines 484-507) and bind them to a governed `chainId`; `verdict-link.schema.json` binds a Chio policy verdict (vocabulary aligned with `spec/schemas/chio-http/v1/verdict.schema.json` and `StepVerdictKind` in `crates/chio-core-types/src/plan.rs` lines 110-138) back to the provenance graph via `requestId`, optional `receiptId`, and `chainId`. NOTE: `stamp.schema.json`, `attestation-bundle.schema.json`, and `verdict-link.schema.json` are prose-drafted because no dedicated `ProvenanceStamp`, `AttestationBundle`, or `VerdictLink` Rust struct exists on this branch yet; the live cross-doc invariant is that M07 (provider-native adapters, phase that wires `chio-tool-call-fabric` to the provenance graph) materializes the matching Rust types for `stamp` and `verdict-link` and re-pins those schemas to the resulting serde shapes, while M09 (supply-chain attestation, phase 3 attestation-verify) materializes the matching Rust type for `attestation-bundle` and re-pins that schema. Field-name casing follows the convention of the closest live Rust shape: `stamp` uses snake_case to match the existing `RuntimeAttestationEvidence` provenance-adjacent shape; `context`, `attestation-bundle`, and `verdict-link` use camelCase to match the `GovernedCallChainContext` family they bind to. No vector cases exist under `tests/bindings/vectors/provenance/` yet; vectors land in M01 P2.
