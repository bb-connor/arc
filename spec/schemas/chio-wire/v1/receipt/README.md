# Chio Wire Schemas v1: receipt/

This subtree contains the machine-readable JSON Schema (draft 2020-12) contract for the receipt primitives that anchor `spec/PROTOCOL.md` section 4 ("Receipts and the Audit Log"). The two schemas land together because they form the auditable evidence record produced by the Kernel: `record.schema.json` describes a single signed receipt envelope (mirroring `ChioReceipt` in `crates/chio-core-types/src/receipt.rs`, with its embedded `Decision`, `ToolCallAction`, and `GuardEvidence` shapes), and `inclusion-proof.schema.json` describes a single Merkle inclusion artifact proving a receipt's membership in the receipt-log tree (mirroring `MerkleProof` in `crates/chio-core-types/src/merkle.rs`). Both schemas are validated against every case in `tests/bindings/vectors/receipt/v1.json` on every PR, and M04 deterministic-replay consumes them as the golden-bundle contract for `tests/replay/goldens/<family>/<name>/`.
