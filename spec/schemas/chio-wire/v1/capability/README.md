# Chio Wire Schemas v1: capability/

This subtree contains the machine-readable JSON Schema (draft 2020-12) contract for the capability primitives that make up the core authority model in `spec/PROTOCOL.md` section 3 ("Capabilities, Delegation, Attenuation"). The three schemas land together because they are interdependent: `token.schema.json` describes the canonical issued capability (mirroring `CapabilityToken` in `crates/chio-core-types/src/capability.rs`); `grant.schema.json` describes a single grant carried inside that token's `scope` (mirroring `ToolGrant`, `ResourceGrant`, and `PromptGrant` in the same Rust module via `oneOf`); and `revocation.schema.json` describes a single revocation entry (mirroring `RevocationRecord` in `crates/chio-kernel/src/revocation_store.rs` and acting as the wire-level companion to the existing `chio-wire/v1/kernel/capability_revoked.schema.json` notification). The schemas are validated against every case in `tests/bindings/vectors/capability/v1.json` on every PR.
