# Chio Wire Schemas v1: jsonrpc/

This subtree contains the machine-readable JSON Schema (draft 2020-12) contract for the JSON-RPC 2.0 wire framing that Chio uses for both MCP and A2A peer transports. Three schemas land together because they form the complete JSON-RPC envelope contract: `request.schema.json` (correlated calls, requires `id` and `method`), `response.schema.json` (success or error reply, requires `id` plus exactly one of `result` or `error` via `oneOf`), and `notification.schema.json` (fire-and-forget message, requires `method`, forbids `id`). The schemas mirror the inline `serde_json::json!` shapes in `crates/chio-mcp-adapter/src/transport.rs` (functions `send_request`, `send_notification`, `json_rpc_result`, `json_rpc_error` at lines 643-648, 770-774, 1299-1316) and the typed `A2aJsonRpcRequest<T>`, `A2aJsonRpcResponse<T>`, `A2aJsonRpcError` structs in `crates/chio-a2a-adapter/src/protocol.rs` (lines 234-255). They are wire-compatible with JSON-RPC 2.0 (https://www.jsonrpc.org/specification). The only Chio-specific extensions are observable in the `error.code` registry (Chio adds -32002 for nested-flow policy denials and -32042 for URL elicitations required, alongside the standard reserved codes and MCP's -32800 for cancellation - see `map_nested_flow_error_code` in transport.rs lines 1280-1297) and the recognized notification method names (Chio adds `CHIO_TOOL_STREAMING_NOTIFICATION_METHOD` for tool output streaming on top of the MCP set). Field shapes themselves remain canonical JSON-RPC 2.0.
