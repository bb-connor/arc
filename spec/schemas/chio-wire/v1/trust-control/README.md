# Chio Wire Schemas v1: trust-control/

This subtree contains the machine-readable JSON Schema (draft 2020-12) contract for the four trust-control plane primitives that anchor `spec/PROTOCOL.md` section 9 ("Trust-Control Contract"). The four schemas land together because they form the lifecycle envelope of one operator-visible authority lease: `lease.schema.json` describes a single authority lease projection (mirroring `ClusterAuthorityLeaseView` in `crates/chio-cli/src/trust_control/service_types.rs` lines 1837-1848, constructed by `cluster_authority_lease_view_locked` in `crates/chio-cli/src/trust_control/cluster_and_reports.rs` lines 841-862); `heartbeat.schema.json` describes a lease refresh request that targets that lease (drafted from prose, see NOTE below); `terminate.schema.json` describes a voluntary lease release request (drafted from prose, see NOTE below); and `attestation.schema.json` describes one normalized runtime attestation evidence statement carried alongside trust-control authority operations and governed capability issuance (mirroring `RuntimeAttestationEvidence` in `crates/chio-core-types/src/capability.rs` lines 484-507, populated by the verifier bridges in `crates/chio-control-plane/src/attestation.rs`). NOTE: `heartbeat.schema.json` and `terminate.schema.json` are drafted from PROTOCOL.md section 9 prose and the lease lifecycle paths in `cluster_and_reports.rs` (lines 832-877 for the lease refresh policy bound and lines 1595-1611 for the leader-loss invalidation path); there are no dedicated `LeaseHeartbeatRequest` or `LeaseTerminateRequest` Rust structs in the live trust-control surface yet, so wire field names follow the `serde(rename_all = camelCase)` convention used by `ClusterAuthorityLeaseView` to keep the family consistent on the wire. The dedicated request/response structs are expected to land alongside the cluster RPC formalization in M09 P3, and these schemas will be re-pinned to those live serde shapes at that time. No vector cases exist under `tests/bindings/vectors/trust-control/` yet; vectors land in M01 P2.
