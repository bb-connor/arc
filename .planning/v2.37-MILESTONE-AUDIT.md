# Milestone Audit: v2.37 arc-settle On-Chain Settlement, Escrow, and Bond Runtime

## Status

Milestone `v2.37` meets its implementation definition of done. Phases 157
through 160 are implemented, verified, and close ARC's first bounded
settlement runtime over the official web3 contract family.

The resulting posture is explicit:

- ARC now ships one real `arc-settle` runtime over escrow create, Merkle-proof
  release, dual-signature release, timeout refund, and bond lifecycle calls
- ARC now projects finality, timeout, reorg, reversal, and recovery posture
  back into canonical web3 settlement receipts instead of hiding that state in
  mutable local bookkeeping
- ARC now ships one bounded Solana-native Ed25519 settlement-preparation lane
  with explicit cross-lane commitment parity rather than claiming ambient
  multi-chain equivalence
- ARC now closes the milestone with qualification, runbook, and public-boundary
  documentation that keep custody and regulated-role assumptions explicit
