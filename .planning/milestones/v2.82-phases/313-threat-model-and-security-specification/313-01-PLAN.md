# Plan 313-01

1. Add a standalone security specification under `spec/` that defines the
   agent-kernel-tool trust boundary, protected assets, attacker goals, and the
   mandatory threat set from the roadmap.
2. For each required threat, map the current shipped controls and any planned
   hardening work, then state the remaining residual risk explicitly.
3. Link the wire-protocol document to the new security spec so protocol readers
   can find the threat model from the normative transport surface.
