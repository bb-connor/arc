# v2.81 Roadmap Snapshot

## v2.81 Deployable ARC and Developer Onboarding

**Status:** complete locally 2026-04-13
**Goal:** Make ARC usable by developers who are not building from Rust source.
Resolve the naming split, publish stable SDKs, provide a one-command Docker
experience, and write a progressive tutorial that takes a developer from zero
to a governed tool call in five minutes.

## Phases

- Phase `307`: Identity Resolution and Scaffolding -- complete
- Phase `308`: SDK Publication -- complete
- Phase `309`: Deployable Experience -- complete
- Phase `310`: Progressive Tutorial and Framework Integration -- complete

## Key Outcomes

- The public onboarding surface is ARC-first and `arc init` scaffolds a working
  governed hello-world project.
- `@arc-protocol/sdk` and `arc-sdk` are both qualified as stable public
  packages with typed clients and governed examples.
- `docker compose up` now boots the hosted edge, trust service, and receipt
  viewer in one local quickstart.
- The progressive tutorial and the Anthropic, LangChain, and OpenAI-compatible
  examples all run against the supported hosted-edge topology.
