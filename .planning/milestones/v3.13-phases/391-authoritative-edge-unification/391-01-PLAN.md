# Plan 391-01

1. Rework `KernelCapabilityChecker` in `arc-acp-proxy` so ACP filesystem and
   terminal authorization routes through the shared `CrossProtocolOrchestrator`
   and kernel receipt pipeline instead of bespoke local token verification.
2. Thread the resulting authorization receipt reference into ACP audit context
   so observed `session/update` receipts can correlate to the authoritative
   live-path decision.
3. Quarantine A2A and ACP direct passthrough helpers behind explicit
   compatibility wrappers, removing them from the default public edge surface.
4. Tighten compatibility and preview metadata so non-authoritative flows are
   machine-readable and cannot satisfy receipt-bearing claim semantics.
5. Update the affected crate tests to prove the new authority path and the
   explicit compatibility-only boundary.

## Status

Completed on 2026-04-14. This plan replaced ACP token-only checking with a
kernel-backed orchestrated guard path, moved A2A/ACP passthrough APIs behind
explicit compatibility wrappers, and tightened non-authoritative metadata.
