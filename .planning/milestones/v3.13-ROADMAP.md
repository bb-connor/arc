# v3.13 Roadmap Snapshot

## v3.13 Universal Orchestration Closure

**Status:** complete locally 2026-04-14; archived locally 2026-04-14
**Goal:** Close the remaining post-v3.12 runtime and truth gaps by landing a
shared cross-protocol orchestrator substrate, authoritative edge unification,
truthful fidelity/publication semantics, HTTP authority/evidence convergence,
truthful lifecycle surfaces, and the first executable cross-protocol runtime
claim gate.

## Key Outcomes

- `arc-cross-protocol` now provides the shared orchestrator substrate and
  receipt-lineage model for authoritative A2A and ACP execution.
- A2A and ACP discovery/publication semantics now use truthful
  `Lossless` / `Adapted` / `Unsupported` fidelity states instead of implying
  more than the runtime actually ships.
- Governed HTTP surfaces (`arc-api-protect` and `arc-tower`) now bind final
  receipt status truthfully and preserve OpenAPI/runtime policy overrides.
- The repo now ships an executable cross-protocol runtime qualification gate
  and machine-readable qualification matrix for the strongest honest
  cross-protocol claim ARC can currently defend.
- Historical milestone/archive truth was improved substantially, while the
  final ledger/archive closeout lane was made explicit and carried into
  `v3.14/401` instead of being left as implicit debt.
