# Plan 379-03

## Objective

Close `OPER-03` by making the Kubernetes admission controller validate ARC
capability tokens against configured ARC trust anchors rather than accepting
self-declared issuers or annotation presence alone.

## Tasks

1. Keep the existing Go controller admission seam in `sdks/k8s/controller`, but
   add explicit trust-anchor configuration so token validation uses one or more
   configured ARC issuer keys.
2. Fail closed when trusted issuer configuration is absent or malformed.
3. Continue validating canonicalized Ed25519 signatures, token time bounds, and
   required scope coverage after trusted issuer resolution succeeds.
4. Extend the admission tests to cover allow, invalid-signature, expired-token,
   out-of-scope, untrusted-issuer, and missing-trust-config paths.
5. Update the platform docs so the runtime contract states the trusted-issuer
   configuration requirement honestly.

## Verification

- `go test ./...` in `sdks/k8s/controller`

## Acceptance

- `OPER-03` is satisfied only when the controller rejects tokens from
  untrusted issuers instead of trusting the key embedded inside the presented
  token.
- Missing or malformed trusted issuer configuration fails closed for
  non-exempt workloads.
- Required scope matching continues to operate over the same token after
  trusted issuer validation succeeds.

## Status

Completed on 2026-04-14. The admission controller now requires configured ARC
trusted issuer keys, validates signatures and time bounds against that trust
anchor, and rejects untrusted or misconfigured workloads fail-closed.
