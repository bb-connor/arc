   Include doc comments (triple-slash `///`) on every type and field:

   ```wit
   package arc:guard@0.1.0;

   /// The verdict a guard returns after evaluating a tool-call request.
   variant verdict {
       /// The guard allows the request to proceed.
       allow,
       /// The guard denies the request with a human-readable reason.
       deny(string),
   }

   /// Read-only request context provided to the guard by the host.
   record guard-request {
       /// Tool being invoked.
       tool-name: string,
       /// Server hosting the tool.
       server-id: string,
       /// Agent making the request.
       agent-id: string,
       /// Tool arguments as a JSON-encoded string.
       arguments: string,
       /// Capability scopes granted (serialized scope names).
       scopes: list<string>,
       /// Host-extracted action type (e.g. "file_access", "network_egress").
       action-type: option<string>,
       /// Normalized file path for filesystem actions.
       extracted-path: option<string>,
       /// Target domain string for network egress actions.
       extracted-target: option<string>,
       /// Session-scoped filesystem roots from the kernel context.
       filesystem-roots: list<string>,
       /// Index of the matched grant in the capability scope.
       matched-grant-index: option<u32>,
   }

   /// The world a guard component targets.
   world guard {
       /// Evaluate a tool-call request and return a verdict.
       export evaluate: func(request: guard-request) -> verdict;
   }
   ```

   carries opaque JSON. `matched-grant-index` is `u32` (not usize) because WIT

1. Update `crates/arc-wasm-guards/src/host.rs` -- modify `create_shared_engine()` to
   enable Component Model support. Add this line BEFORE `Engine::new(&config)`:

   ```rust
   config.wasm_component_model(true);
   ```

   This is additive: core modules still work on an Engine with component model enabled.
   The `wasm_component_model(true)` call is required for `Component::new()` to succeed.
   Without it, Component::new() returns "component model support is not enabled".

2. Create `crates/arc-wasm-guards/src/component.rs` gated behind `#[cfg(feature = "wasmtime-runtime")]`.
   This module contains:

   a) The bindgen! invocation. The path is relative to the crate's Cargo.toml:
   ```rust
   wasmtime::component::bindgen!({
       path: "../../wit/arc-guard",
       world: "guard",
   });
   ```
   This generates: a `Guard` struct with `instantiate()` and `call_evaluate()`,
   a `Verdict` enum, and a `GuardRequest` struct. These live in the module's namespace.

   b) A `ComponentBackend` struct that implements `WasmGuardAbi`:
   ```rust
   pub struct ComponentBackend {
       engine: Arc<Engine>,
       component: Option<wasmtime::component::Component>,
       fuel_limit: u64,
       max_memory_bytes: usize,
       max_module_size: usize,
       last_fuel_consumed: Option<u64>,
   }
   ```

   c) Constructor methods:
   - `pub fn with_engine(engine: Arc<Engine>) -> Self` (default limits)
   - `pub fn with_limits(mut self, max_memory_bytes: usize, max_module_size: usize) -> Self` (builder)

   d) `impl WasmGuardAbi for ComponentBackend`:

   **load_module:** Check `wasm_bytes.len() > self.max_module_size` (same WGSEC-03
   pattern as WasmtimeBackend). Then call `wasmtime::component::Component::new(&self.engine, wasm_bytes)`
   mapping errors to `WasmGuardError::Compilation`. Store in `self.component`.

   **evaluate:** 
   - Get component ref or return `WasmGuardError::BackendUnavailable`.
   - Create a fresh `Store::new(&self.engine, ())` -- guard world has no imports,
     so `Store<()>` suffices (no WasmHostState needed for component path).
   - Set fuel: `store.set_fuel(self.fuel_limit)`.
   - Apply StoreLimits for memory capping: create a `StoreLimitsBuilder::new()
     .memory_size(self.max_memory_bytes).trap_on_grow_failure(true).build()`,
     store it alongside the store data. Since Store<()> does not carry state,
     use a wrapper tuple struct: `struct ComponentState(StoreLimits)` and
     `Store::new(&self.engine, ComponentState(limits))` with
     `store.limiter(|s| &mut s.0)`.
   - Create `wasmtime::component::Linker::<ComponentState>::new(&self.engine)` (empty, no imports).
   - Instantiate: `Guard::instantiate(&mut store, &component, &linker)`.
   - Convert `crate::abi::GuardRequest` to the bindgen-generated `GuardRequest` via `to_wit_request()`.
   - Call `bindings.call_evaluate(&mut store, &wit_request)`.
   - Track fuel: `store.get_fuel().unwrap_or(0)` -> compute consumed.
   - Convert bindgen `Verdict` to `crate::abi::GuardVerdict` via `from_wit_verdict()`.

   **backend_name:** Return `"wasmtime-component"`.
   **last_fuel_consumed:** Return `self.last_fuel_consumed`.

   e) Two private conversion functions:

   `fn to_wit_request(req: &crate::abi::GuardRequest) -> GuardRequest`:
   - Field-by-field clone. `arguments` -> `serde_json::to_string(&req.arguments).unwrap_or_default()`.
   - `matched_grant_index` -> `req.matched_grant_index.map(|i| i as u32)`.
   - All other fields: `.clone()`.

   `fn from_wit_verdict(v: Verdict) -> crate::abi::GuardVerdict`:
   - `Verdict::Allow => GuardVerdict::Allow`
   - `Verdict::Deny(reason) => GuardVerdict::Deny { reason: Some(reason) }`

   IMPORTANT constraints:
   - Do NOT use `unwrap()` or `expect()` anywhere (clippy deny). Use `.map_err()` 
     with `WasmGuardError` variants, or `.unwrap_or()` / `.unwrap_or_default()`.
   - The entire module must be gated: `#[cfg(feature = "wasmtime-runtime")]`.
   - Use fully qualified paths (`crate::abi::GuardRequest`) to avoid name collision
     with the bindgen-generated `GuardRequest`.
   - Do NOT add `use crate::abi::GuardRequest` at the module top level.
