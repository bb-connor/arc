# Plan 66-01

Add a reference holder adapter that can consume ARC verifier requests and
produce standards-native presentation responses.

## Scope

- build a minimal holder helper around the ARC SD-JWT VC profile
- support request fetch, response assembly, and verifier submission
- keep the adapter explicitly bounded to qualification and partner integration

## Success Criteria

- ARC ships one usable reference holder path for demos and tests
- the holder adapter does not claim to be a general wallet product
- unsupported request shapes fail closed with clear errors
