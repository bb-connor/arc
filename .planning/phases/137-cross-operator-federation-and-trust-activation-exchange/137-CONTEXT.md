# Phase 137: Cross-Operator Federation and Trust-Activation Exchange - Context

## Goal

Define cross-operator federation artifacts so trust activation can be shared without becoming ambient trust.

## Why This Phase Exists

Federation begins with explicit exchanged activation contracts, not with mirrors or reputation scores.

## Scope

- federation contracts
- trust-activation exchange
- scope and delegation rules
- local import and attenuation semantics

## Out of Scope

- quorum and anti-eclipse mechanics
- shared-reputation clearing
- wallet-network interoperability
