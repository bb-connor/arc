# Plan 65-01

Define ARC's narrow OID4VP verifier profile and signed request-object
contract.

## Scope

- choose the primary verifier transport profile for ARC
- define request-object fields, signing, and verifier transaction identity
- constrain the supported credential and claim-request surface to ARC SD-JWT VC

## Success Criteria

- ARC has one explicit OID4VP verifier contract instead of ad hoc challenge
  semantics
- unsupported client identifier schemes and credential families fail closed
- the request-object profile is stable enough for storage and HTTP transport
