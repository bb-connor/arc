# Plan 65-03

Implement OID4VP response handling and verifier-side validation for the ARC
SD-JWT VC profile.

## Scope

- accept `direct_post.jwt` responses
- verify request binding, nonce, audience, disclosure selection, and status
- return explicit verifier results without relying on ARC-native holder
  challenge artifacts

## Success Criteria

- ARC can verify one standards-native presentation response end to end
- replay, stale, mismatched, or over-disclosed responses fail closed
- regression coverage exists for the supported and unsupported response paths
