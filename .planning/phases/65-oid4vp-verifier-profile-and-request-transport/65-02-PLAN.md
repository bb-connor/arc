# Plan 65-02

Implement replay-safe verifier transaction storage plus request transport.

## Scope

- persist verifier transactions with nonce, state, and expiration boundaries
- publish signed requests through `request_uri`
- add same-device and cross-device launch artifacts over the same verifier
  transaction state

## Success Criteria

- ARC can create and retrieve verifier requests by reference
- QR and redirect flows reuse the same canonical verifier transaction truth
- expired or replayed verifier transactions fail closed
