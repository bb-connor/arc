# Plan 95-01

Define ARC's bounded request-time authorization contract.

## Scope

- map governed request semantics into request-time authorization details
- specify how approval tokens and capabilities relate to hosted authorization artifacts
- keep canonical receipts authoritative even when live hosted auth is present

## Success Criteria

- ARC can describe governed requests in standards-facing terms before execution
- approval and execution truth remain separable and auditable
- request-time and post-execution semantics are consistent
