# Plan 161-01

Implement the bounded EVM-side proof-verification fallback path.

- add Functions request and response types
- prepare bounded batch verification requests from signed ARC receipts
- assess returned DON results against the prepared request

## Success Criteria

- ARC can prepare one bounded receipt batch for Functions verification
- the runtime rejects mismatched request, chain, root, or size data
- the fallback remains explicit about accepted versus rejected outcomes
