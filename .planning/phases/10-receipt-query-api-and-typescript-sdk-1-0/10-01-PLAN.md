    - test_query_filter_time_range_both: since+until combined returns only receipts in window
    - test_query_filter_cost_range_min: min_cost excludes non-financial receipts and receipts with cost < min_cost

    - test_query_filter_cost_range_both: min_cost+max_cost combined returns only receipts in cost window
    - test_query_cursor_pagination: cursor=N returns only receipts with seq > N
    - test_query_cursor_pagination_pages: paginating through results with cursor yields all receipts without duplicates
    - test_query_next_cursor_some_when_more: next_cursor is Some(last_seq) when results.len() == limit
    - test_query_next_cursor_none_when_last_page: next_cursor is None when results.len() < limit
    - test_query_total_count: total_count reflects filtered count independent of limit/cursor
    - test_query_limit_capped: limit above MAX_QUERY_LIMIT is capped to MAX_QUERY_LIMIT
    - test_query_combined_filters: multiple filters applied simultaneously (capability_id + outcome + time range)
  </behavior>
  <action>
    Create `crates/arc-kernel/src/receipt_query.rs`:

    1. Define `MAX_QUERY_LIMIT` constant (200, matching existing MAX_LIST_LIMIT in trust_control.rs).

    2. Define `ReceiptQuery` struct with fields per CONTEXT.md locked decision:
       - capability_id: Option<String>
       - tool_server: Option<String>
       - tool_name: Option<String>
       - outcome: Option<String> (maps to decision_kind column)
       - since: Option<u64> (Unix seconds, inclusive)
       - until: Option<u64> (Unix seconds, inclusive)
       - min_cost: Option<u64> (minor units)
       - max_cost: Option<u64> (minor units)
       - cursor: Option<u64> (seq to start after, exclusive)
       - limit: usize (page size, capped at MAX_QUERY_LIMIT)
       Derive Debug, Default, Clone.

    3. Define `ReceiptQueryResult` struct:
       - receipts: Vec<StoredToolReceipt>
       - total_count: u64
       - next_cursor: Option<u64>
       Derive Debug.

    4. Implement `query_receipts` as a public method on `SqliteReceiptStore` (not a free function -- keeps `connection` private). The method signature:
       `pub fn query_receipts(&self, query: &ReceiptQuery) -> Result<ReceiptQueryResult, ReceiptStoreError>`

    5. SQL query uses the IS NULL OR parameterized pattern from list_tool_receipts, extended with:
       - `(?5 IS NULL OR timestamp >= ?5)` for since
       - `(?6 IS NULL OR timestamp <= ?6)` for until
       - `(?7 IS NULL OR CAST(json_extract(raw_json, '$.metadata.financial.cost_charged') AS INTEGER) >= ?7)` for min_cost
       - `(?8 IS NULL OR CAST(json_extract(raw_json, '$.metadata.financial.cost_charged') AS INTEGER) <= ?8)` for max_cost
       - `(?9 IS NULL OR seq > ?9)` for cursor
       - ORDER BY seq ASC, LIMIT ?10

    6. Run a separate COUNT(*) query with the same WHERE clause (no ORDER BY, no LIMIT, no cursor filter) to get total_count. The count query should NOT include the cursor filter because total_count represents the full filtered dataset size.

    7. Compute next_cursor: if receipts.len() == limit (capped), set to Some(last receipt's seq). Otherwise None.

    8. In `crates/arc-kernel/src/lib.rs`:
       - Add `pub mod receipt_query;` after `pub mod receipt_store;`
       - Add `pub use receipt_query::{ReceiptQuery, ReceiptQueryResult, MAX_QUERY_LIMIT};` to the re-exports

    Note: Tests need receipts with financial metadata. Create helper functions that build ArcReceipt with FinancialReceiptMetadata populated in the metadata field, using the same signing pattern as sample_receipt_with_id in receipt_store.rs tests.
