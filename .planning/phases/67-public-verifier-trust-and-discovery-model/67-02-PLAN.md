# Plan 67-02

Implement verifier trust publication and rotation semantics.

## Scope

- publish verifier trust material at stable HTTPS locations
- define key and certificate rotation behavior
- keep verifier trust material aligned with the chosen request transport

## Success Criteria

- holders and external wallets can retrieve verifier trust material
- verifier rotation does not silently break active trust assumptions
- stale or mismatched trust material fails closed
