## Phase Boundary

Correct the product boundary after phases `183` and `184`: ARC remains the
generic substrate, while MERCURY moves to a dedicated app surface built on top
of ARC instead of shipping as an `arc` subcommand.

<decisions>
## Implementation Decisions

### Mercury Stays Opinionated
- keep proof, inquiry, and pilot workflow semantics in MERCURY-owned code
- ship the user-facing command surface as `crates/arc-mercury`

### ARC Stays Generic
- keep evidence export and control-plane helpers reusable from ARC crates
- remove MERCURY-specific command wiring and dependencies from `arc-cli`

### Correct The Ledger Too
- update pilot/verifier docs to use `arc-mercury` and `mercury`
- amend roadmap, state, and milestone audit so they describe the corrected
  architecture rather than the superseded coupling

</decisions>
