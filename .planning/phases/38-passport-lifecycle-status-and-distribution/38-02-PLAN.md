# Phase 38 Plan 02

## Objective

Implement passport lifecycle management and verifier-side status enforcement
across CLI and trust-control surfaces.

## Must Haves

- Operators can publish lifecycle changes through supported CLI or
  trust-control flows.
- Verifiers can enforce lifecycle state instead of treating every valid signed
  passport as perpetually current.
- Lifecycle checks compose with the existing multi-issuer passport model.
