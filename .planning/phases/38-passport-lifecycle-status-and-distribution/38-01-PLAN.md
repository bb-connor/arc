# Phase 38 Plan 01

## Objective

Define the passport lifecycle model, including status, revocation,
supersession, and supported distribution semantics.

## Must Haves

- There is one explicit lifecycle state model for active, revoked, and
  superseded passports.
- Distribution metadata is sufficient for relying parties and verifiers to
  fetch or cache lifecycle state.
- The lifecycle model preserves historical verification truth instead of
  rewriting old signed objects.
