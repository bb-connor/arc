**WAT module builder helpers:**

1. `build_trivial_guard_wat() -> &'static str` -- Returns a minimal WAT module
   that imports all three arc.* host functions, exports memory (1 page) and
   `evaluate` returning `i32.const 0` (VERDICT_ALLOW). Approximately 200 bytes
   compiled. Used for instantiation benchmarks.

2. `build_sized_wat(target_binary_bytes: usize) -> String` -- Builds a WAT module
   with a `(data ...)` segment padded with `\00` bytes to approximate the target
   compiled binary size. The function should:
   - Start with the same arc.* imports as trivial guard
   - Add a data segment with `\00` repeated `target_binary_bytes - 300` times
     (subtracting approximate WAT overhead for imports/exports/header)
   - Set `(memory (export "memory") N)` where N = (target_binary_bytes / 65536) + 1
   - Export `evaluate` returning `i32.const 0`
   - Note: WAT data segments produce roughly 1:1 binary size. Accept approximate
     sizes within ~10% of target.

**Benchmark group 1: `bench_module_compilation` (WGBENCH-01)**
```rust
fn bench_module_compilation(c: &mut Criterion) {
    let engine = create_shared_engine().unwrap();

    let small_wat = build_sized_wat(50 * 1024);  // ~50 KiB target
    let large_wat = build_sized_wat(5 * 1024 * 1024);  // ~5 MiB target

    let mut group = c.benchmark_group("wasm_guards/compilation");

    group.bench_function("50kib_module", |b| {
        b.iter(|| {
            let _ = black_box(Module::new(black_box(&engine), black_box(small_wat.as_bytes())));
        });
    });

    // Increase sample size consideration for slow compilations
    group.sample_size(10);
    group.bench_function("5mib_module", |b| {
        b.iter(|| {
            let _ = black_box(Module::new(black_box(&engine), black_box(large_wat.as_bytes())));
        });
    });

    group.finish();
}
```

The 50 KiB benchmark validates against the 50ms threshold from the decision doc.
The 5 MiB benchmark documents large-module compilation time (may exceed 50ms; that
is expected and should be noted).

**Benchmark group 2: `bench_instantiation` (WGBENCH-02)**
```rust
fn bench_instantiation(c: &mut Criterion) {
    let engine = create_shared_engine().unwrap();
    let trivial_wat = build_trivial_guard_wat();
    let module = Module::new(&engine, trivial_wat).unwrap();

    let mut group = c.benchmark_group("wasm_guards/instantiation");

    group.bench_function("trivial_guard", |b| {
        b.iter_batched(
            || {
                // Setup: pre-build linker with host functions (matches isolating
                // instantiation cost from linker setup)
                let host_state = WasmHostState::new(HashMap::new());
                let mut store = Store::new(&engine, host_state);
                store.limiter(|s| &mut s.limits);
                store.set_fuel(10_000_000).unwrap();
                let mut linker = Linker::new(&engine);
                register_host_functions(&mut linker).unwrap();
                (store, linker)
            },
            |(mut store, linker)| {
                // Measured: instantiation only
                let _instance = black_box(
                    linker.instantiate(&mut store, &module).unwrap()
                );
            },
            criterion::BatchSize::SmallInput,
        );
    });

    group.finish();
}
```

This measures pure `linker.instantiate()` per-call overhead with a pre-compiled
module. The setup closure creates fresh Store + Linker each iteration (matching
production behavior where each evaluate() gets a fresh Store), but the module
compilation is excluded from measurement.
