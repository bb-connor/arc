          - Load via create_backend, evaluate with `make_request("read_file")`
          - Assert `verdict.is_allow()`

          - Evaluate with `make_request("dangerous_tool")`
          - Assert `verdict.is_deny()`
          - Assert deny reason contains "blocked by policy"

       e. `ts_guard_denies_rm_rf`:
          - Evaluate with `make_request("rm_rf")`
          - Assert `verdict.is_deny()`

       f. `ts_guard_denies_drop_database`:
          - Evaluate with `make_request("drop_database")`
          - Assert `verdict.is_deny()`

    max_module_size on ComponentBackend. If `create_backend()` returns
    `WasmGuardError::ModuleTooLarge`, the test should document this and either:
    (a) use `ComponentBackend::with_engine().with_limits()` directly instead of
