# Plan 150-02

Integrate cross-currency denial and approval behavior into receipt-side
artifacts.

## Scope

- attach `financial.oracle_evidence` on successful conversion
- record explicit receipt-side conversion status details
- mark failed settlement paths when cross-currency reconciliation cannot be justified

## Success Criteria

- approval receipts preserve successful oracle provenance
- failed cross-currency reconciliation is visible in receipt metadata
- provisional charges remain conservative when conversion fails
