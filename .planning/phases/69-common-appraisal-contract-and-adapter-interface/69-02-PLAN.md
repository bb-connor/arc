# Plan 69-02

Define the verifier-adapter interface and registration boundary.

## Scope

- specify adapter input and output contracts
- define verifier identity and adapter metadata requirements
- make freshness, replay, and evidence-reference semantics uniform

## Success Criteria

- ARC can add verifier families without changing policy semantics ad hoc
- adapter outputs are comparable only through the canonical appraisal contract
- unsupported adapters fail closed instead of degrading silently
