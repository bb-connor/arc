# Plan 69-01

Define the canonical ARC appraisal contract.

## Scope

- introduce typed appraisal structures and status semantics
- separate verifier identity, raw evidence references, normalized assertions,
  and vendor-scoped claims
- define stable reason codes for failure, degradation, and unsupported inputs

## Success Criteria

- ARC has one auditable appraisal schema instead of adapter-specific blobs
- normalized assertions are explicit and bounded
- malformed or incomplete appraisal artifacts fail closed
