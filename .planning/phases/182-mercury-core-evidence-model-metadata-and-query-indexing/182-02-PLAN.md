# Plan 182-02

Extend SQLite persistence with extracted MERCURY business-identifier indexes so
pilot investigation flows do not rely on raw JSON scans.

## Scope

- add a dedicated MERCURY receipt-index table keyed by `receipt_id`
- extract business identifiers and approval/decision state when receipts are
  appended
- backfill and query the index safely for existing receipt rows

## Success Criteria

- workflow, account, desk, strategy, release, rollback, exception, and inquiry
  identifiers are indexed explicitly
- approval-state and decision-type filters use extracted columns instead of
  production JSON scans
- malformed MERCURY metadata fails closed instead of creating partial indexes
