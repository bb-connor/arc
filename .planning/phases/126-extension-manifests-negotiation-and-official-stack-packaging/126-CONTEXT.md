# Phase 126: Extension Manifests, Negotiation, and Official Stack Packaging - Context

## Goal

Define machine-readable extension manifests, capability negotiation, versioning, and one official ARC stack package over first-party components.

## Why This Phase Exists

Once extension points are known, ARC needs a compatibility contract and a default packaged stack before third-party implementations are allowed in.

## Scope

- extension manifest schema
- capability and version negotiation
- official ARC stack package layout
- fail-closed mismatch handling

## Out of Scope

- runtime sandboxing for custom code
- third-party qualification matrices
- web3-specific contract work
