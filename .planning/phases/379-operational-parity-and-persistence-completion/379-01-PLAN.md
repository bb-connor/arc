# Plan 379-01

1. Add a lightweight SQLite-backed `HttpReceipt` persistence layer to
   `arc-api-protect` that activates when `ProtectConfig.receipt_db` is set.
2. Persist receipts from both the proxy handler and `/arc/evaluate`, and fail
   closed if configured persistence cannot be written.
3. Reload persisted receipts into the in-memory receipt log on startup so
   receipt history survives state recreation.
4. Add focused tests proving configured receipt persistence works for proxy
   flows and remains visible when the state is recreated for sidecar
   evaluation.

## Status

Completed and independently re-verified on 2026-04-14. The SQLite persistence
slice is landed in `arc-api-protect`, and the phase remains active only for the
remaining `arc-tower` body-binding and Kubernetes admission slices.
