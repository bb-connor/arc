# Plan 54-01

Define the portable lifecycle contract for issued ARC passports.

## Scope

- add typed status, revocation, supersession, and distribution-reference
  contracts that project from existing lifecycle truth
- define how ARC maps `active`, `revoked`, and `superseded` into
  external-compatible status semantics without losing richer ARC meaning
- validate all portable lifecycle documents fail closed on contradictory or
  incomplete state

## Success Criteria

- ARC has one explicit typed portable status contract for issued passports
- supersession remains explicit and auditable rather than being silently
  collapsed into revocation or omitted
- distribution references are structurally valid and tied back to the issuer's
  lifecycle truth surface
