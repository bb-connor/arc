# Phase 238 Context

With the motion and boundary frozen, Mercury can now define one activation
package rooted in the broader-distribution truth chain. This phase turns that
bounded lane into a typed contract and Mercury CLI export surface.

The implementation must remain Mercury-specific. It can add one new core
contract and one new `arc-mercury` export path, but it must not imply generic
onboarding tooling, CRM workflows, channel marketplaces, or ARC-side
commercial surfaces.
