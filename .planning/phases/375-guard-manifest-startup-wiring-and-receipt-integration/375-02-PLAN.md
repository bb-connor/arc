       ```rust
       /// Load WASM guards from WasmGuardEntry configs with manifest verification.
       ///
       /// For each entry:
       /// 1. Loads `guard-manifest.yaml` from the .wasm file's parent directory
       /// 2. Validates abi_version against SUPPORTED_ABI_VERSIONS
       /// 3. Reads the .wasm binary and verifies SHA-256 against manifest
       /// 4. Creates a WasmtimeBackend with the manifest's config
       /// 5. Loads the module into the backend
       /// 6. Creates a WasmGuard with the manifest SHA-256 for receipt metadata
       ///
       /// Entries are sorted by priority (lower first) before loading.
       /// Non-advisory guards are loaded before advisory guards at equal priority.
       pub fn load_wasm_guards(
           entries: &[WasmGuardEntry],
           engine: Arc<Engine>,
       ) -> Result<Vec<WasmGuard>, WasmGuardError>
       ```

       Implementation details:
       - Clone entries, sort by `(entry.advisory as u8, entry.priority)` so non-advisory
         come first at equal priority, and lower priority values come first
       - For each sorted entry:
         a. Call `manifest::load_manifest(&entry.path)` to get `GuardManifest`
         b. Call `manifest::verify_abi_version(&manifest.abi_version)`
         c. Read wasm bytes via `std::fs::read(&entry.path)`, map error to
            `WasmGuardError::ModuleLoad`
         d. Call `manifest::verify_wasm_hash(&wasm_bytes, &manifest.wasm_sha256)`
         e. Compute the manifest SHA-256 for receipt metadata: this is the SHA-256 of the
            manifest YAML file content itself (NOT the wasm binary -- the wasm hash is in
            the manifest and verified separately). Wait -- re-reading requirement WGRCPT-02:
            "the guard manifest SHA-256 hash is recorded" -- this refers to the wasm_sha256
            from the manifest (the .wasm binary's hash). Store `manifest.wasm_sha256.clone()`
            as the `manifest_sha256` on WasmGuard.
         f. Create backend: `WasmtimeBackend::with_engine_and_config(engine.clone(), manifest.config.clone())`
         g. Set limits: `.with_limits(entry.max_memory_bytes_or_default, entry.max_module_size_or_default)`.
            Note: `WasmGuardEntry` from arc-config does NOT have max_memory_bytes or
            max_module_size fields. Use defaults from WasmtimeBackend (no `.with_limits()` call
            needed -- the defaults are already correct). If in the future these fields are
            added to WasmGuardEntry, the wiring can be updated.
         h. Call `backend.load_module(&wasm_bytes, entry.fuel_limit)?`
         i. Create `WasmGuard::new(entry.name.clone(), Box::new(backend), entry.advisory, Some(manifest.wasm_sha256.clone()))`
       - Return the vec of WasmGuards

       /// Build a complete guard pipeline with the correct tier ordering.

       pub fn build_guard_pipeline(
