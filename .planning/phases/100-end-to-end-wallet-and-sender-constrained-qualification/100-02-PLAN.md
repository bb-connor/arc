# Plan 100-02

Qualify asynchronous exchange and sender-constrained failures.

## Scope

- one asynchronous or message-oriented exchange path
- DPoP, mTLS, and attestation-bound negative cases
- fail-closed handling for stale, mismatched, or replayed sender proof

## Success Criteria

- asynchronous exchange is qualified explicitly
- sender-constrained failures are tested directly
- unsupported sender semantics stay bounded
