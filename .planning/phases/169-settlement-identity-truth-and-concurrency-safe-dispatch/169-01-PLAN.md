# Plan 169-01

Replace nonce-derived settlement identity with deterministic contract truth.

## Scope

- add `deriveEscrowId` and `deriveVaultId` to the official contract family
- derive escrow and bond identity from immutable term content instead of
  mutable nonce state
- update the Rust bindings and runtime preflight path to consume the new
  identity surface

## Success Criteria

- interleaving submissions no longer change escrow or vault identity
- runtime preflight no longer depends on stateful `eth_call` against mutating
  contract methods
- contract artifacts and Rust bindings stay aligned
