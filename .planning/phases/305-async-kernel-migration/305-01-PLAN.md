# Plan 305-01

## Objective

Convert the kernel’s core runtime state from direct mutable ownership to
interior mutability so operational APIs can stop requiring `&mut self`.

## Tasks

1. Move mutable runtime fields behind `Mutex`, `RwLock`, and atomics.
2. Convert session/read/queue helpers to work through shared-access closures or
   cloned snapshots instead of returning direct mutable borrows.
3. Convert receipt/checkpoint recording helpers to shared-state access.

## Acceptance

- The kernel compiles with interior-mutability-backed runtime state.
- Operational helpers no longer depend on an exclusive kernel borrow.
- No behavior changes are introduced in receipt or session semantics.
