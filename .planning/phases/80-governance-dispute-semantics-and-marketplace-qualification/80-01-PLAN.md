# Plan 80-01

Define governance and dispute semantics for public certification state.

## Scope

- specify dispute, supersession, revocation, and evidence-update workflows
- define who can publish changes and how provenance is preserved
- keep governance visible without turning ARC into a trust oracle

## Success Criteria

- governance state is explicit and auditable
- disputes and evidence updates have stable semantics
- public listing state remains provenance-preserving
