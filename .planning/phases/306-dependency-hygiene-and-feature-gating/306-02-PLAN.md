# Plan 306-02

## Objective

Feature-gate the alloy/web3 stack so the core workspace build can exclude EVM
runtime crates while preserving the default web3-enabled behavior.

## Tasks

1. Add explicit `web3` features to the alloy-heavy crates and keep them enabled
   by default.
2. Make `arc-link` partially available without `web3` by swapping in stub
   Chainlink/sequencer modules.
3. Gate `arc-anchor`, `arc-settle`, and `arc-web3-bindings` at the crate level
   so they compile as empty crates when `web3` is disabled.
4. Verify the normal `--no-default-features` workspace graph excludes alloy,
   ethers, Solidity, and duplicate `reqwest`/`serde_yaml`/`hashbrown` headers.

## Acceptance

- `cargo build --workspace --no-default-features` succeeds.
- `cargo tree -e normal --no-default-features` contains no alloy-family crates.
- `cargo tree -e normal -d --no-default-features` has no duplicate headers for
  `reqwest`, `serde_yaml`, or `hashbrown`.
