# Plan 172-01

Wire cryptographic secondary-lane verification into proof-bundle validation.

## Scope

- validate Bitcoin/OpenTimestamps secondary lanes against the ARC super-root
  digest and declared block attestation
- reject undeclared secondary-lane evidence in proof bundles
- align core proof validation and public docs with the hardened bundle rules

## Success Criteria

- Bitcoin secondary-lane verification fails closed on digest, block, or lane
  mismatch
- proof bundles reject extra undeclared secondary evidence
- ARC's public anchor boundary claims cryptographic secondary-lane validation
  rather than metadata-presence checks
