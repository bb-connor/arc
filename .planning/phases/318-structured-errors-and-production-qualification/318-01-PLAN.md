# Plan 318-01

1. Introduce a serializable structured error-report surface in `arc-kernel`
   and `arc-control-plane` that exposes `code`, `context`, and
   `suggested_fix`.
2. Add a global `--format json` selector to `arc-cli`, keep `--json`
   compatible, and route top-level failure rendering through the new report
   helpers.
3. Add targeted unit tests for kernel/CLI report mapping, JSON rendering, and
   CLI parsing so the first phase slice is mechanically verified.
