# Phase 119: Trust Activation Artifacts and Open Admission Classes - Context

## Goal

Define trust-activation artifacts and open admission classes so registry
visibility never collapses into runtime admission.

## Why This Phase Exists

ARC's endgame can widen publication and discovery, but it still must preserve
local operator control over what becomes trusted or executable.

## Scope

- trust-activation artifacts and review records
- open admission classes and bounded eligibility semantics
- separation of listing visibility from local trust activation
- fail-closed handling for missing, stale, or incompatible activation state

## Out of Scope

- governance charters or dispute case artifacts
- portable reputation exchange and weighting
- final endgame qualification and release closure
