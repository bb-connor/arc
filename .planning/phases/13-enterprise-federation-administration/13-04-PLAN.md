    Add explicit operator-facing administration and debugging surfaces:

    1. In `crates/arc-cli/src/main.rs`, add `arc trust provider` subcommands:
       - `list`
       - `get --provider-id <id>`
       - `upsert --input <json-file>`
       - `delete --provider-id <id>`

    2. In `crates/arc-cli/src/trust_control.rs`, add authenticated HTTP routes:
       - `GET /v1/federation/providers`
       - `GET /v1/federation/providers/{provider_id}`
       - `PUT /v1/federation/providers/{provider_id}`
       - `DELETE /v1/federation/providers/{provider_id}`

    3. Route behavior must reuse the file-backed registry from 13-01:
       - list/get return provider records with `validation_errors`, provenance metadata, and trust-boundary metadata
       - upsert recalculates validation immediately before saving
       - delete removes the record from the registry file

    4. Surface enterprise diagnostics in existing operator/debug outputs:
       - when `enterprise_audit` exists on federated issue allow responses, include it in CLI JSON output without renaming fields
       - when enterprise admission denies, ensure the structured error or response body includes provider id, provider record id, trust material reference, matched/missing organization/group/role context, and decision reason when safe to expose
       - surface source-attribute provenance for canonical principal, tenant, organization, groups, and roles so operators can trace which provider-native fields produced the normalized output
       - extend session/admin trust output in `mcp_serve_http` flows to include `enterpriseIdentity` consistently when enterprise federation is active

    5. Add integration tests:
       - `provider_admin.rs`: registry upsert/list/get/delete over CLI and/or HTTP
       - `provider_admin.rs`: invalid provider records stay listable with provenance/trust-boundary validation failures visible
       - `mcp_serve_http.rs`: admin trust output exposes `enterpriseIdentity`, source provenance, and the legacy-vs-enterprise lane boundary truthfully
       - `federated_issue.rs`: allow and deny responses expose structured enterprise diagnostics and do not regress legacy bearer-only admission
  </action>
