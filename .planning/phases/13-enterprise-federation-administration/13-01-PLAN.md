Output: file-backed provider registry types, validation rules, and CLI/config plumbing shared by `pact mcp serve-http` and `pact trust serve`.

    - .planning/phases/13-enterprise-federation-administration/13-CONTEXT.md

  </read_first>
  <action>
    Create `crates/pact-cli/src/enterprise_federation.rs` with these exact core types and behaviors:

    1. Define `EnterpriseProviderKind` enum with variants:
       - `OidcJwks`
       - `OauthIntrospection`
       - `Scim`
       - `Saml`

    2. Define `EnterpriseSubjectMapping` struct with these fields:
       - `principal_source: String`
       - `client_id_field: Option<String>`
       - `object_id_field: Option<String>`
       - `tenant_id_field: Option<String>`
       - `organization_id_field: Option<String>`
       - `groups_field: Option<String>`
       - `roles_field: Option<String>`

    3. Define `EnterpriseProviderRecord` with these fields:
       - `provider_id: String`
       - `kind: EnterpriseProviderKind`
       - `enabled: bool`
       - `provenance: EnterpriseProviderProvenance`
       - `trust_boundary: EnterpriseTrustBoundary`
       - `provider_profile: Option<crate::remote_mcp::JwtProviderProfile>`
       - `issuer: Option<String>`
       - `discovery_url: Option<String>`
       - `jwks_url: Option<String>`
       - `introspection_url: Option<String>`
       - `scim_base_url: Option<String>`
       - `saml_entity_id: Option<String>`
       - `saml_metadata_url: Option<String>`
       - `tenant_id: Option<String>`
       - `organization_id: Option<String>`
       - `subject_mapping: EnterpriseSubjectMapping`
       - `validation_errors: Vec<String>`

    3a. Define `EnterpriseProviderProvenance` with these fields:
       - `configured_from: String`
       - `source_ref: Option<String>`
       - `trust_material_ref: Option<String>`
       - `subject_mapping_source: Option<String>`

    3b. Define `EnterpriseTrustBoundary` with these fields:
       - `allowed_issuers: Vec<String>`
       - `allowed_audiences: Vec<String>`
       - `allowed_tenants: Vec<String>`
       - `allowed_organizations: Vec<String>`

    4. Define `EnterpriseProviderRegistry` as:
       - `version: String`
       - `providers: BTreeMap<String, EnterpriseProviderRecord>`
       Use JSON as the on-disk format. Version string is exactly `"pact.enterprise-providers.v1"`.

    5. Implement `EnterpriseProviderRecord::validate(&self) -> Vec<String>` with kind-specific rules:
       - `OidcJwks`: require `issuer` and at least one of `discovery_url` or `jwks_url`
       - `OauthIntrospection`: require `issuer` and `introspection_url`
       - `Scim`: require `scim_base_url`
       - `Saml`: require `saml_entity_id` and `saml_metadata_url`
       - all kinds require non-empty `provider_id`
       - all kinds require non-empty `provenance.configured_from`
       - all kinds require non-empty `provenance.trust_material_ref`
       - all kinds require non-empty `subject_mapping.principal_source`
       - if `issuer` is present and `trust_boundary.allowed_issuers` is non-empty, the issuer must appear in that allowlist
       - if `tenant_id` is present and `trust_boundary.allowed_tenants` is non-empty, the tenant must appear in that allowlist
       - if `organization_id` is present and `trust_boundary.allowed_organizations` is non-empty, the organization must appear in that allowlist
       - if `enabled` is true and validation errors exist, the provider is treated as invalid and unavailable for admission

    6. Implement registry helpers:
       - `EnterpriseProviderRegistry::load(path: &Path) -> Result<Self, CliError>`
       - `EnterpriseProviderRegistry::save(&self, path: &Path) -> Result<(), CliError>`
       - `EnterpriseProviderRegistry::validated_provider(&self, provider_id: &str) -> Option<&EnterpriseProviderRecord>`
       - `EnterpriseProviderRegistry::upsert(&mut self, record: EnterpriseProviderRecord)`
       - `EnterpriseProviderRegistry::remove(&mut self, provider_id: &str) -> bool`

    7. Validation behavior must be operator-visible and fail-closed:
       - validation errors are stored on the record
       - invalid records remain listable for diagnostics
       - provenance and trust-boundary failures are reported as explicit validation errors rather than silently ignored
       - `validated_provider` returns `None` for disabled or invalid records

    8. Add unit tests in the same module covering:
       - OIDC provider missing issuer is invalid
       - introspection provider missing introspection_url is invalid
       - SCIM provider missing base URL is invalid
       - SAML provider missing metadata URL is invalid
       - provider with missing provenance trust-material reference is invalid
       - provider with out-of-bound issuer or tenant is invalid
       - registry save/load round-trips without losing validation state
       - invalid enabled record is excluded by `validated_provider`
  </action>
