    **Step 3: Extend ToolGrant::is_subset_of** to check monetary fields. Add after the existing constraints check (before `true` return):

    For max_cost_per_invocation: if parent has Some, child must have Some with child.units <= parent.units AND same currency. If currencies differ, return false.

    For max_total_cost: same logic -- if parent has Some, child must have Some with child.units <= parent.units AND same currency.

    The subset check logic mirrors the existing max_invocations pattern:
    ```rust
    // If parent has a per-invocation cost cap, child must too and it must be <=
    if let Some(ref parent_cost) = parent.max_cost_per_invocation {
        match &self.max_cost_per_invocation {
            Some(child_cost) if child_cost.currency == parent_cost.currency
                && child_cost.units <= parent_cost.units => {}
            _ => return false,
        }
    }

    // If parent has a total cost cap, child must too and it must be <=
    if let Some(ref parent_cost) = parent.max_total_cost {
        match &self.max_total_cost {
            Some(child_cost) if child_cost.currency == parent_cost.currency
                && child_cost.units <= parent_cost.units => {}
            _ => return false,
        }
    }
    ```
