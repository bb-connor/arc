1. Implement `cmd_guard_pack() -> Result<(), CliError>`:
   - Read `guard-manifest.yaml` from the current directory. Use `std::fs::read_to_string("guard-manifest.yaml")` and parse with `serde_yml::from_str::<GuardManifest>()` (import the struct from arc_wasm_guards if accessible, or define a minimal local struct if the import path is complex -- the manifest shape is simple: name, version, abi_version, wasm_path, wasm_sha256, config).
   - Verify the wasm_path from the manifest exists on disk. Read the .wasm bytes.
   - Create the archive filename: `{name}-{version}.arcguard`
   - Create a gzipped tar archive using `flate2::write::GzEncoder` wrapping a `std::fs::File`, then `tar::Builder` wrapping the encoder:
     ```rust
     use flate2::write::GzEncoder;
     use flate2::Compression;

     let archive_name = format!("{}-{}.arcguard", manifest.name, manifest.version);
     let file = fs::File::create(&archive_name).map_err(|e| {
         CliError::Other(format!("failed to create {archive_name}: {e}"))
     })?;
     let enc = GzEncoder::new(file, Compression::default());
     let mut tar = tar::Builder::new(enc);

     // Add guard-manifest.yaml
     tar.append_path("guard-manifest.yaml").map_err(|e| {
         CliError::Other(format!("failed to add manifest to archive: {e}"))
     })?;

     // Add the .wasm file
     tar.append_path(&manifest.wasm_path).map_err(|e| {
         CliError::Other(format!("failed to add wasm to archive: {e}"))
     })?;

     tar.into_inner().map_err(|e| {
         CliError::Other(format!("failed to finalize archive: {e}"))
     })?.finish().map_err(|e| {
         CliError::Other(format!("failed to finish gzip: {e}"))
     })?;
     ```
   - Print: `packed: {archive_name} ({formatted_size})`
   - NOTE: `tar.append_path` preserves the relative path inside the archive. The .wasm path from the manifest might be something like `target/wasm32-unknown-unknown/release/my_guard.wasm`. Inside the archive, store only the filename (not the full relative path). Use `tar.append_file(filename, &mut file)` with a custom `tar::Header` instead of `append_path` to control the archive entry name. Store entries as: `guard-manifest.yaml` and `{underscored_name}.wasm` (the .wasm filename component only).

2. Implement `cmd_guard_install(archive_path: &Path, target_dir: &Path) -> Result<(), CliError>`:
   - Open the archive file, wrap in `flate2::read::GzDecoder`, then `tar::Archive`:
     ```rust
     use flate2::read::GzDecoder;

     let file = fs::File::open(archive_path).map_err(|e| {
         CliError::Other(format!("failed to open {}: {e}", archive_path.display()))
     })?;
     let dec = GzDecoder::new(file);
     let mut archive = tar::Archive::new(dec);
     ```
   - Extract all entries. The archive contains `guard-manifest.yaml` and the `.wasm` file.
   - Before extracting, parse the manifest entry from the archive to get the guard name, then create a subdirectory `{target_dir}/{guard_name}/`.
   - Extract both files into `{target_dir}/{guard_name}/`.
   - After extraction, update the `wasm_path` field in the extracted `guard-manifest.yaml` to point to the co-located .wasm filename (since the original wasm_path may have been a build-time relative path like `target/.../foo.wasm`).
   - Print: `installed: {guard_name} to {target_dir}/{guard_name}/`
   - A simpler approach that avoids two-pass archive reading: extract all entries to a temp dir first, read the manifest from temp, determine guard name, create final dir, move files. This avoids seeking issues with gzip streams.

3. Add unit tests:

Call `cmd_guard_pack()` (with current dir set appropriately -- or test the internal packing logic directly by extracting it into a helper that takes paths). Verify the .arcguard file exists and is > 0 bytes. Then call `cmd_guard_install` with the archive, verify extracted files exist and content matches.
   - `test_pack_fails_without_manifest`: No guard-manifest.yaml in dir -> CliError
   - `test_pack_fails_with_missing_wasm`: Manifest references .wasm that does not exist -> CliError
   - `test_install_fails_with_missing_archive`: Archive path does not exist -> CliError

   For tests that need to change the current working directory, use helper functions that take explicit paths rather than relying on `std::env::set_current_dir` (which is process-global and breaks parallel tests). Refactor pack logic to accept a `project_dir: &Path` parameter internally, with the public `cmd_guard_pack()` calling it with `"."`.
