# Plan 377-01

1. Extend `MessageInterceptor` so `fs/read_text_file`, `fs/write_text_file`,
   and `terminal/create` extract an additive capability token from ACP params,
   call the installed `CapabilityChecker`, and fail closed on deny or checker
   error before the built-in guards run.
2. Upgrade `KernelCapabilityChecker` to reject untrusted issuers and invalid
   token signatures before the existing time-bound and scope checks.
3. Propagate the validated `capability_id` into ACP audit entries and
   `KernelReceiptSigner` so receipt-like artifacts distinguish
   cryptographically enforced operations from audit-only observations.
4. Add focused `arc-acp-proxy` tests covering checker invocation, deny/error
   fail-closed behavior, signature verification, and capability metadata
   propagation.
