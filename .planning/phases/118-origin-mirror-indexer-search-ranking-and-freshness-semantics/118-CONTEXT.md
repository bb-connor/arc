# Phase 118: Origin, Mirror, Indexer, Search, Ranking, and Freshness Semantics - Context

## Goal

Define how origin operators, mirrors, indexers, search, ranking, and freshness
metadata behave over the generic registry substrate.

## Why This Phase Exists

Open publication only works if ARC also makes freshness, replication, and
ranking explicit and reproducible instead of implicit implementation detail.

## Scope

- origin, mirror, and indexer roles
- freshness and transparency metadata
- search, ranking, and reproducibility semantics
- fail-closed behavior for stale, divergent, or unverifiable registry state

## Out of Scope

- trust activation and admission classes
- governance charters and disputes
- portable reputation and fee economics
