# Plan 183-01

Wrap verified ARC evidence export output into `Proof Package v1` without
forking ARC's canonical receipt, checkpoint, or inclusion-proof truth.

## Scope

- define MERCURY publication-profile and proof-package contracts in
  `arc-mercury-core`
- bind the proof package to a verified ARC evidence export manifest hash
- require bundle manifests plus typed MERCURY receipt metadata for every
  exported receipt

## Success Criteria

- `Proof Package v1` is a strict wrapper over verified ARC evidence export
- package validation fails closed on schema, workflow-scope, or manifest-bind
  mismatch
- the contract is reusable by the CLI verifier path and later pilot harness
