# Phase 394 Plan 01

## Objective

Make the Rust HTTP lane one coherent, truthful authority story instead of a
cluster of similar-but-divergent surfaces.

## Work Items

1. Decide whether `response_status` becomes real downstream evidence,
   evaluation-time semantics, or a split field model, then implement and
   document that choice across runtime and schema surfaces.
2. Thread `arc-openapi` runtime override metadata into `arc-api-protect`
   evaluation so explicit side-effect and approval annotations beat method
   defaults.
3. Expand proxy header forwarding to an operator-credible default allowlist
   with tests and explicit exclusions.
4. Converge `arc-tower` onto the same authority/evidence path as the sidecar,
   or explicitly narrow what it claims to provide.
