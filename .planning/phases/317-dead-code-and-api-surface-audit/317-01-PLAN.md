# Plan 317-01

1. Audit every non-test `#[allow(dead_code)]` site and decide whether the code
   should be removed or retained with an explicit justification comment.
2. Apply the lowest-risk fixes first across feature-gated stubs, envelope
   deserializers, and helper methods that are intentionally kept for parity.
3. Re-run targeted formatting and compile/test commands on the touched crates
   so the dead-code cleanup does not introduce regressions.
