# Plan 316-01

1. Replace the single cached `rusqlite::Connection` in `SqliteReceiptStore`
   with an `r2d2_sqlite` pool configured with the existing WAL and busy-timeout
   pragmas.
2. Move the hot runtime write path onto pooled connections:
   `append_arc_receipt_returning_seq`, `append_child_receipt`,
   `record_capability_snapshot`, and `store_checkpoint`.
3. Add one concurrent-write regression test that proves a shared store instance
   can accept parallel receipt inserts without duplicate-seq or lock errors.
