# Phase 28 Plan 01

## Objective

Reduce the remaining large domain files to thin facades over named source
files.

## Must Haves

- `arc-credentials`, `arc-reputation`, and `arc-policy` no longer hide most
  of their behavior in one file each
- the new file boundaries compile without caller-visible API churn
- `arc-policy` uses a nested `evaluate/` folder instead of one monolithic
  `evaluate.rs`
