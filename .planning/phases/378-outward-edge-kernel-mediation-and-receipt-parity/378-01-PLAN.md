# Plan 378-01

1. Add explicit kernel-backed A2A send and JSON-RPC execution helpers that turn
   `SendMessage` requests into `ToolCallRequest` values and return signed
   receipt metadata in the resulting task surface.
2. Add explicit kernel-backed ACP invocation helpers plus capability-aware
   permission preview so the outward ACP surface can emit signed receipts and
   truthful authorization decisions when capability context is available.
3. Reword direct passthrough helper comments in both crates so they are clearly
   described as non-kernel compatibility paths, not the primary ARC trust path.
4. Add focused regression tests for allow/deny receipt emission and capability
   preview behavior in both edge crates.

## Status

Completed and re-verified on 2026-04-14. The default outward A2A/ACP entrypoints
now route through the kernel, and the former direct paths remain only as
explicitly named non-authoritative compatibility helpers.
