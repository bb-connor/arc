# Plan 170-01

Make receipt persistence and checkpoint issuance explicit prerequisites for
web3-enabled runtime activation.

## Scope

- add policy and kernel config for `require_web3_evidence`
- reject activation and startup when receipt persistence is absent
- reject append-only receipt stores that cannot issue kernel-signed
  checkpoints

## Success Criteria

- web3-enabled deployments fail closed without a durable receipt store
- disabling checkpoint issuance blocks web3 activation
- local SQLite receipt persistence remains accepted as the bounded supported
  substrate
