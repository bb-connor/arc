<objective>
Create the `arc-guard-sdk-macros` proc-macro crate that provides the `#[arc_guard]` attribute macro. The macro transforms a single user-written `fn evaluate(req: GuardRequest) -> GuardVerdict` into a complete WASM guard binary with all ABI exports: the `evaluate` extern "C" entry point, `arc_alloc`/`arc_free` allocator re-exports, and the `arc_deny_reason` glue re-export.

Purpose: Eliminate all manual ABI boilerplate for guard authors. One attribute macro replaces ~20 lines of unsafe glue code per guard.
Output: New `crates/arc-guard-sdk-macros/` crate added to workspace, implementing GSDK-06.
</objective>

   - Import `proc_macro::TokenStream`, `quote::quote`, `syn::{parse_macro_input, ItemFn}`
   - Define `#[proc_macro_attribute] pub fn arc_guard(_attr: TokenStream, item: TokenStream) -> TokenStream`
   - Parse input as `ItemFn`, extract: `sig.ident` (user fn name), `block` (body), `sig.inputs` (params), `sig.output` (return type), `attrs` (doc comments etc), `vis` (visibility)

     d. `#[no_mangle] pub extern "C" fn evaluate(ptr: i32, len: i32) -> i32` that:
        - Calls `unsafe { arc_guard_sdk::read_request(ptr, len) }` in a match
        - On `Err(_)` returns `arc_guard_sdk::VERDICT_DENY` (fail-closed)
        - On `Ok(r)` calls the renamed user function with `r`
        - Passes the verdict to `arc_guard_sdk::encode_verdict(verdict)` and returns the result
     e. Apply `#[allow(clippy::cast_possible_truncation, clippy::cast_possible_wrap)]` to the generated evaluate function

3. Add `"crates/arc-guard-sdk-macros"` to the `[workspace] members` array in the root `Cargo.toml`
