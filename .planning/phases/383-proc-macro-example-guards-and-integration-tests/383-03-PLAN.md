pub trait WasmGuardAbi: Send + Sync {

    pub extracted_target: Option<String>,
    pub filesystem_roots: Vec<String>,

}

pub enum GuardVerdict {

   - `use arc_wasm_guards::runtime::wasmtime_backend::WasmtimeBackend;`

4. Helper function `load_example_wasm(name: &str) -> Vec<u8>`:
   - Build path: `format!("{}/../../target/wasm32-unknown-unknown/release/{}.wasm", env!("CARGO_MANIFEST_DIR"), name)`
   - Read the file with `std::fs::read(&path).unwrap_or_else(|e| panic!("Missing .wasm at {path}: {e}. Build with: cargo build --target wasm32-unknown-unknown --release -p <example-crate>"))`

5. Helper function `make_request(tool_name: &str) -> GuardRequest`:
   - Returns a GuardRequest with the given tool_name, server_id="test-server", agent_id="test-agent", arguments=json!({}), and all optional fields as None/empty

6. Helper function `make_enriched_request(tool_name: &str, action_type: Option<&str>, extracted_path: Option<&str>) -> GuardRequest`:
   - Same as make_request but sets action_type and extracted_path from params

7. Tests for tool-gate guard:

   a. `tool_gate_allows_safe_tool`: Load `arc_example_tool_gate.wasm`, create WasmtimeBackend with shared engine, load_module with fuel_limit 1_000_000, evaluate with tool_name="read_file". Assert `verdict.is_allow()`.

   b. `tool_gate_denies_dangerous_tool`: Same setup, evaluate with tool_name="dangerous_tool". Assert `verdict.is_deny()`. Check reason contains "blocked by policy" via match on GuardVerdict::Deny { reason: Some(r) } where r contains "blocked by policy".

   c. `tool_gate_denies_rm_rf`: Same setup, evaluate with tool_name="rm_rf". Assert `verdict.is_deny()`.

   d. `tool_gate_denies_drop_database`: Same setup, evaluate with tool_name="drop_database". Assert `verdict.is_deny()`.

8. Tests for enriched-inspector guard:

   a. `enriched_inspector_allows_non_file_write`: Load `arc_example_enriched_inspector.wasm`, create WasmtimeBackend with shared engine, load_module with fuel_limit 1_000_000, evaluate with make_request("any_tool"). Assert `verdict.is_allow()`.

   b. `enriched_inspector_allows_file_read`: Same setup, evaluate with make_enriched_request("read_file", Some("file_access"), Some("/etc/passwd")). Assert `verdict.is_allow()` (action is file_access, not file_write).

   c. `enriched_inspector_denies_write_to_etc`: Same setup, evaluate with make_enriched_request("write_file", Some("file_write"), Some("/etc/passwd")). Assert `verdict.is_deny()`. Check reason contains "write to /etc blocked".

   d. `enriched_inspector_allows_write_to_tmp`: Same setup, evaluate with make_enriched_request("write_file", Some("file_write"), Some("/tmp/data.txt")). Assert `verdict.is_allow()`.

   e. `enriched_inspector_denies_write_to_configured_path`: Create WasmtimeBackend with `with_engine_and_config(engine, HashMap::from([("blocked_path".to_string(), "/var/secret".to_string())]))`, load_module, evaluate with make_enriched_request("write_file", Some("file_write"), Some("/var/secret/key.pem")). Assert `verdict.is_deny()`. Check reason contains "protected path".

For each test, create a FRESH WasmtimeBackend instance (the WasmtimeBackend uses per-call Store, but Module loading is one-time). Use `create_shared_engine().unwrap()` to get a shared engine for all tests.
