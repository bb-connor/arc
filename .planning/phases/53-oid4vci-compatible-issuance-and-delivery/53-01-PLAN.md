# Plan 53-01

## Objective

Define the OID4VCI-compatible issuer metadata, offer, token, and credential
contracts that map cleanly onto ARC passport issuance without changing ARC's
credential trust model.

## Work

- add typed portable issuance profile documents in `arc-credentials`
- define validation rules for issuer metadata, pre-authorized offer grants,
  token exchange, and delivered passport payloads
- keep `did:arc`, Ed25519-signed credentials, and ARC passport bundle truth as
  the underlying credential model

## Verification

- unit coverage for interoperable issuance contract validation
- regression coverage proving malformed metadata, grants, or requests fail
  closed
