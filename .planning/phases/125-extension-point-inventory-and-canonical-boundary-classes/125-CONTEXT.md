# Phase 125: Extension-Point Inventory and Canonical Boundary Classes - Context

## Goal

Define ARC's named extension points and classify which surfaces are canonical truth versus replaceable adapters.

## Why This Phase Exists

The extension stack only works if ARC first freezes which contracts are authoritative and which seams are safely replaceable.

## Scope

- extension-point inventory across ARC crates
- canonical truth versus replaceable surface classes
- stability tiers and ownership boundaries
- trust-boundary rules for extension seams

## Out of Scope

- machine-readable manifest formats
- official stack packaging
- qualification execution
