# Plan 125-02

Define canonical truth classes versus replaceable adapter classes.

## Scope

- mark receipt, policy, trust, and artifact surfaces that stay canonical
- mark adapters and providers that remain replaceable
- document which extension classes can supply evidence or execution but not redefine truth

## Success Criteria

- canonical and pluggable surfaces are clearly separated
- extension classes cannot silently redefine ARC truth
- the boundary is machine-reviewable enough for later manifest work
