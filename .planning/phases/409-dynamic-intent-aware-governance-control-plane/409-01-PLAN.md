# Phase 409 Plan

## Objective

Turn the shared protocol fabric into a real intent-aware control plane by
making route choice dynamic, signed, and visible across the claim-eligible
surfaces.

## Execution

1. Add route-planning types that distinguish candidate availability,
   selection, attenuation, and deny outcomes.
2. Make `CrossProtocolOrchestrator` plan routes from intent, policy,
   capability, and availability rather than passive metadata alone.
3. Surface signed route-selection evidence on the authoritative HTTP, MCP,
   OpenAI, A2A, and ACP paths.
4. Add regression coverage for select, attenuate, and deny outcomes.
