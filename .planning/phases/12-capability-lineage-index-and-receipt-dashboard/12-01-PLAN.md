1. Add capability_lineage table DDL and indexes to the execute_batch in SqliteReceiptStore::open (receipt_store.rs), after the kernel_checkpoints table block:

```sql
CREATE TABLE IF NOT EXISTS capability_lineage (
    capability_id        TEXT PRIMARY KEY,
    subject_key          TEXT NOT NULL,
    issuer_key           TEXT NOT NULL,
    issued_at            INTEGER NOT NULL,
    expires_at           INTEGER NOT NULL,
    grants_json          TEXT NOT NULL,
    delegation_depth     INTEGER NOT NULL DEFAULT 0,
    parent_capability_id TEXT REFERENCES capability_lineage(capability_id)
);
CREATE INDEX IF NOT EXISTS idx_capability_lineage_subject
    ON capability_lineage(subject_key);
CREATE INDEX IF NOT EXISTS idx_capability_lineage_issued_at
    ON capability_lineage(issued_at);
CREATE INDEX IF NOT EXISTS idx_capability_lineage_parent
    ON capability_lineage(parent_capability_id);
```

2. Create crates/pact-kernel/src/capability_lineage.rs with:

- `CapabilitySnapshot` struct (capability_id, subject_key, issuer_key, issued_at, expires_at, grants_json as String, delegation_depth, parent_capability_id as Option<String>). Derive Debug, Clone, Serialize, Deserialize. Do NOT use deny_unknown_fields (SCHEMA-01).

- `CapabilityLineageError` enum with Sqlite(rusqlite::Error) and Json(serde_json::Error) variants via thiserror.

- `record_capability_snapshot` method on SqliteReceiptStore:
  - Accepts `&mut self`, `token: &CapabilityToken`, `parent_capability_id: Option<&str>`
  - Serializes `token.scope` to JSON via serde_json::to_string
  - Computes delegation_depth: if parent_capability_id is Some, query the parent's depth and add 1; if None, depth is 0
  - Uses `INSERT OR IGNORE INTO capability_lineage` (idempotent on re-insert)
  - Returns `Result<(), CapabilityLineageError>`

- `get_lineage` method on SqliteReceiptStore:
  - Accepts `&self`, `capability_id: &str`
  - Returns `Result<Option<CapabilitySnapshot>, CapabilityLineageError>`
  - Simple SELECT by PRIMARY KEY

- `get_delegation_chain` method on SqliteReceiptStore:
  - Accepts `&self`, `capability_id: &str`
  - Returns `Result<Vec<CapabilitySnapshot>, CapabilityLineageError>`
  - Uses WITH RECURSIVE CTE:
    ```sql
    WITH RECURSIVE chain(..., level) AS (
        SELECT ..., 0 FROM capability_lineage WHERE capability_id = ?1
        UNION ALL
        SELECT cl.*, chain.level + 1
        FROM capability_lineage cl
        INNER JOIN chain ON cl.capability_id = chain.parent_capability_id
        WHERE chain.level < 20
    )
    SELECT * FROM chain ORDER BY level DESC
    ```
  - Returns root-first ordering (ORDER BY level DESC -- deepest ancestor has highest level, comes first)

- `list_capabilities_for_subject` method on SqliteReceiptStore:
  - Accepts `&self`, `subject_key: &str`
  - Returns `Result<Vec<CapabilitySnapshot>, CapabilityLineageError>`
  - SELECT * FROM capability_lineage WHERE subject_key = ?1 ORDER BY issued_at DESC

3. Add `pub mod capability_lineage;` to lib.rs. Re-export `CapabilitySnapshot` and `CapabilityLineageError`.

4. Important: use `pub(crate)` for the impl methods since they need access to `self.connection` (the private Connection field on SqliteReceiptStore). The pattern already exists in receipt_store.rs for query_receipts_impl.
