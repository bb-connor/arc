Extend the receipt query API with agent-centric filtering via JOIN and add lineage HTTP endpoints.

2. Modify `query_receipts_impl` in receipt_store.rs:

**Data query SQL change:** Add LEFT JOIN and agent filter parameter. The new SQL becomes:
```sql
SELECT r.seq, r.raw_json
FROM pact_tool_receipts r
LEFT JOIN capability_lineage cl ON r.capability_id = cl.capability_id
WHERE (?1 IS NULL OR r.capability_id = ?1)
  AND (?2 IS NULL OR r.tool_server = ?2)
  AND (?3 IS NULL OR r.tool_name = ?3)
  AND (?4 IS NULL OR r.decision_kind = ?4)
  AND (?5 IS NULL OR r.timestamp >= ?5)
  AND (?6 IS NULL OR r.timestamp <= ?6)
  AND (?7 IS NULL OR CAST(json_extract(r.raw_json, '$.metadata.financial.cost_charged') AS INTEGER) >= ?7)
  AND (?8 IS NULL OR CAST(json_extract(r.raw_json, '$.metadata.financial.cost_charged') AS INTEGER) <= ?8)
  AND (?9 IS NULL OR r.seq > ?9)
  AND (?11 IS NULL OR cl.subject_key = ?11)
ORDER BY r.seq ASC
LIMIT ?10
```

Note: parameter ?11 is the agent_subject filter. Using ?11 (after ?10 limit) to keep parameter ordering clean. Alternatively, renumber so agent_subject is ?9, cursor is ?10, limit is ?11. Choose whichever ordering keeps the code cleanest, but be consistent between data and count queries.

**Count query SQL change:** Same LEFT JOIN and ?N IS NULL OR cl.subject_key = ?N filter, without cursor/limit.
