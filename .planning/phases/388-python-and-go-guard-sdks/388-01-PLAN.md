```wit
interface types {
    variant verdict {
        allow,
        deny(string),
    }
    record guard-request {
        tool-name: string,
        server-id: string,
        agent-id: string,
        arguments: string,
        scopes: list<string>,
        action-type: option<string>,
        extracted-path: option<string>,
        extracted-target: option<string>,
        filesystem-roots: list<string>,
        matched-grant-index: option<u32>,
    }
}
world guard {
    use types.{verdict, guard-request};
    export evaluate: func(request: guard-request) -> verdict;
}
```

<!-- TS SDK pattern to mirror -->

```typescript
const BLOCKED_TOOLS: ReadonlySet<string> = new Set(["dangerous_tool", "rm_rf", "drop_database"]);
export function evaluate(request: GuardRequest): Verdict {
  if (BLOCKED_TOOLS.has(request.toolName)) {
    return { tag: "deny", val: "tool is blocked by policy" };
  }
  return { tag: "allow" };
}
```

Create build scripts and compile the example guard:

1. **scripts/generate-types.sh** (chmod +x): Generate Python bindings from WIT.
   ```bash
   componentize-py -d ../../../wit/arc-guard -w guard bindings .
   ```
   The WIT path is `../../../wit/arc-guard` from `packages/sdk/arc-guard-py/`
   (3 levels deep, same as TS SDK -- verified in Phase 387).

2. **scripts/build-guard.sh** (chmod +x): Full build pipeline:
   - Step 1: Generate bindings (call generate-types.sh or inline the command)
   - Step 2: Create `dist/` directory if missing
   - Step 3: Compile with componentize-py:
     ```bash
     componentize-py \
         -d ../../../wit/arc-guard \
         -w guard \
         componentize \
         --stub-wasi \
         examples/tool-gate/guard \
         -o dist/tool-gate.wasm
     ```
   - Step 4: Print output file size

   CRITICAL: The `--stub-wasi` flag is mandatory. Without it, CPython's stdlib
   will import `wasi:cli` which the ARC host linker cannot satisfy (zero-import
   guard world security model).

3. **Run the build**: Execute `./scripts/build-guard.sh` from the package
   directory. This will:
   - Generate bindings (discover actual module/class names)
   - If the example guard imports don't match the generated names, update
     `examples/tool-gate/guard.py` to use the correct import paths
   - Compile to `dist/tool-gate.wasm`

4. **Verify the output**: Run `wasm-tools component wit dist/tool-gate.wasm`
   (if wasm-tools is available) to confirm the component exports `evaluate`
   and has zero imports.

   If componentize-py is not installed, install it first:
   ```bash
   pip install componentize-py
   ```

   Expected binary size: 10-35 MiB (bundles CPython interpreter).
