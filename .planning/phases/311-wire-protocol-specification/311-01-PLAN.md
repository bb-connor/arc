# Plan 311-01

1. Add a focused wire-protocol specification under `spec/` that defines the
   shipped native ARC transport, including framing, canonical JSON rules,
   message size limits, and framing error recovery.
2. Document the complete native message catalog with explicit field tables for
   each `AgentMessage` and `KernelMessage` variant.
3. Update the broader `spec/PROTOCOL.md` profile so it points readers at the
   new focused wire spec instead of leaving the transport contract implicit.
