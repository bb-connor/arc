    pub filesystem_roots: Vec<String>,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub matched_grant_index: Option<usize>,
}

pub enum GuardVerdict {
    Allow,
    Deny { reason: Option<String> },
}

pub trait WasmGuardAbi: Send + Sync {
    fn load_module(&mut self, wasm_bytes: &[u8], fuel_limit: u64) -> Result<(), WasmGuardError>;
    fn evaluate(&mut self, request: &GuardRequest) -> Result<GuardVerdict, WasmGuardError>;
    fn backend_name(&self) -> &str;
    fn last_fuel_consumed(&self) -> Option<u64> { None }
}

Each loader resolves paths via `env!("CARGO_MANIFEST_DIR")` and `../../` prefix like existing integration tests.

**Fixture loading:**
`load_fixtures(relative_path: &str) -> Vec<TestFixture>` -- reads YAML from `{CARGO_MANIFEST_DIR}/../../tests/conformance/fixtures/guard/{relative_path}`, deserializes via `serde_yml::from_str`.

**Verdict checking:**
`check_verdict(fixture: &TestFixture, verdict: &GuardVerdict) -> Result<(), String>`:
- If expected_verdict is "allow", verdict must be `GuardVerdict::Allow`
- If expected_verdict is "deny", verdict must be `GuardVerdict::Deny { reason }` and if `deny_reason_contains` is set, reason must contain that substring
- Returns `Err(description)` on mismatch

**Main test function:**
`#[test] fn conformance_tool_gate_all_languages()`:
1. Load the tool-gate.yaml fixtures
2. Build the guard registry (all 4 languages)
3. Create a shared `Arc<Engine>` via `create_shared_engine()`
4. For each guard entry:
   a. Try to load the guard. If `None`, print `"[SKIP] {name}: guard WASM not found"` and continue
   b. For each fixture, call `evaluate(&fixture.request)` on a fresh backend instance (create a new backend per fixture to reset fuel state)
   c. Check the verdict. Print `"[PASS] {guard_name} / {fixture_name}"` or `"[FAIL] {guard_name} / {fixture_name}: {reason}"`
   d. Track pass/fail counts
5. After all guards and fixtures, print summary: `"conformance: {passed}/{total} passed, {skipped} guards skipped"`
6. `assert_eq!(failed, 0, "conformance failures detected")` -- fail the test if any fixture failed

**IMPORTANT DETAIL -- fresh backend per fixture:** The runner must create a new backend instance per fixture (not per guard). This is because fuel metering in WasmtimeBackend tracks remaining fuel across calls. Creating a fresh backend per fixture ensures accurate per-fixture fuel readings for CONF-03 in plan 02. The loader functions should return the wasm bytes + a closure that creates a backend, not a pre-loaded backend.

Revised pattern:
```rust
struct GuardEntry {
    name: &'static str,
    wasm_bytes: Vec<u8>,
    make_backend: fn(Arc<Engine>, &[u8]) -> Box<dyn WasmGuardAbi>,
}
```

Where `make_backend` creates a fresh backend, calls `load_module`, and returns it. The `try_load_guard` functions return `Option<GuardEntry>` (None if .wasm missing).
