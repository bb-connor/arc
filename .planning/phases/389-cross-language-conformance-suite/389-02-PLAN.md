<!-- Fuel measurement API from WasmGuardAbi trait -->

From crates/arc-wasm-guards/src/abi.rs:
```rust
pub trait WasmGuardAbi: Send + Sync {
    fn load_module(&mut self, wasm_bytes: &[u8], fuel_limit: u64) -> Result<(), WasmGuardError>;
    fn evaluate(&mut self, request: &GuardRequest) -> Result<GuardVerdict, WasmGuardError>;
    fn backend_name(&self) -> &str;

    /// Return fuel consumed during the last evaluate() call, if tracked.
    /// Backends that support fuel metering return Some(consumed).
    fn last_fuel_consumed(&self) -> Option<u64> {
        None
    }
}
```

The `last_fuel_consumed()` method is available on both `WasmtimeBackend` (core modules)
and `ComponentBackend` (component model). Both return `Some(fuel)` after each
`evaluate()` call when fuel metering is enabled (which it is -- `create_shared_engine()`
enables `consume_fuel(true)` in the wasmtime Config).

Fuel units are wasmtime instruction fuel -- comparable across backends running on
the same engine. However, Component Model modules do more work per evaluate
(component instantiation, canonical ABI lifting/lowering) so they naturally consume
more fuel than core modules even for identical logic.

The 2x threshold is per-fixture across languages: for fixture F, if the minimum
fuel across all guards is `min_fuel`, then every guard must have
`fuel <= 2 * min_fuel` or the test fails.

**Add fuel tracking struct:**
```rust
/// Fuel consumed by a specific guard on a specific fixture.
#[derive(Debug)]
struct FuelRecord {
    guard_name: String,
    fixture_name: String,
    fuel_consumed: u64,
}
```

**Modify the conformance_tool_gate_all_languages test** to:
1. After each successful `evaluate()` call, read `backend.last_fuel_consumed()`
2. If `Some(fuel)`, create a `FuelRecord` and push to a `Vec<FuelRecord>`
3. Print fuel alongside verdict: `"[PASS] {guard} / {fixture} (fuel: {fuel})"` or `"[PASS] {guard} / {fixture} (fuel: N/A)"` if None
4. After the pass/fail loop, call a new function `check_fuel_parity(&fuel_records)` that enforces the 2x threshold

**Implement `check_fuel_parity(records: &[FuelRecord]) -> Result<(), String>`:**
1. Group records by fixture_name into a `HashMap<String, Vec<(String, u64)>>` (fixture -> [(guard_name, fuel)])
2. For each fixture with at least 2 fuel entries:
   a. Find `min_fuel` (the most efficient guard)
   b. Find `max_fuel` (the least efficient guard)
   c. If `max_fuel > 2 * min_fuel`, record a violation: `"{fixture}: {max_guard} used {max_fuel} fuel vs {min_guard}'s {min_fuel} (ratio: {ratio:.1}x, threshold: 2.0x)"`
3. Return `Err` with all violations joined by newline, or `Ok(())` if none

**Print fuel summary table** before the parity check:
```
--- Fuel Summary ---
Fixture              | rust      | typescript | python    | go
allow_safe_tool      | 12345     | 234567     | 345678    | 45678
deny_dangerous_tool  | 11000     | 220000     | 330000    | 44000
...
```

Use right-aligned columns, pad with spaces. If a guard was skipped, show "-" in its column.

**Assert at end of test:**
```rust
if let Err(violations) = check_fuel_parity(&fuel_records) {
    panic!("fuel parity violations:\n{violations}");
}
```

**IMPORTANT:** The 2x threshold compares within the same fixture across languages. It does NOT compare Rust core modules to Component Model modules for the enriched-inspector test (that only has Rust). Only apply fuel parity to `conformance_tool_gate_all_languages` where multiple languages run the same fixtures.

For the `conformance_enriched_inspector_rust` test, still collect and print fuel data but do NOT enforce parity (only one language).
