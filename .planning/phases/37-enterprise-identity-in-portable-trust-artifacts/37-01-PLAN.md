# Phase 37 Plan 01

## Objective

Define one canonical enterprise-identity provenance model for portable
credentials, presentations, and passport artifacts.

## Must Haves

- Passport and presentation artifacts can carry enterprise identity provenance
  without relying on untyped metadata blobs.
- The model distinguishes issuer/subject identity from enterprise-derived
  context so later federation work can attenuate it safely.
- Serialization remains deterministic and compatible with the existing signed
  artifact pipeline.
