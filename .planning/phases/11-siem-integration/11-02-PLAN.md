  <action>
    1. Create `crates/pact-siem/src/exporters/splunk.rs`:

    2. Define `SplunkConfig`:
       - `endpoint: String` -- Splunk HEC endpoint URL (e.g. "https://splunk.example.com:8088")
       - `hec_token: String` -- HEC authentication token
       - `sourcetype: String` -- default "pact:receipt"
       - `index: Option<String>` -- optional Splunk index name (omit to use default index)
       - `host: Option<String>` -- optional host field

    3. Define `SplunkHecExporter`:
       - Fields: `config: SplunkConfig`, `client: reqwest::Client`
       - `SplunkHecExporter::new(config: SplunkConfig) -> Result<Self, ExportError>`:
         Build reqwest::Client with rustls, return Self. Use `reqwest::Client::builder().build()` and map errors to ExportError::HttpError.

    4. Implement `Exporter for SplunkHecExporter`:
       - `name()` returns `"splunk-hec"`
       - `export_batch(&self, events: &[SiemEvent]) -> Result<usize, ExportError>`:
         a. If events is empty, return Ok(0)
         b. Build payload as newline-separated JSON objects (NOT a JSON array):
            Each event envelope: `{"time": <receipt.timestamp as f64>, "sourcetype": "<sourcetype>", "event": <full receipt JSON>}`
            If config.index is Some, add `"index"` field to envelope
            If config.host is Some, add `"host"` field to envelope
         c. Serialize each envelope with serde_json::to_string, join with "\n"
         d. POST to `{endpoint}/services/collector/event`:
            - Header: `Authorization: Splunk {hec_token}`
            - Header: `Content-Type: application/json`
            - Body: the newline-separated payload
         e. Check response status: if not 2xx, return ExportError::HttpError with status code and body text
         f. On success, return Ok(events.len())

    5. Update `crates/pact-siem/src/exporters/mod.rs` to declare `pub mod splunk;`
    6. Update `crates/pact-siem/src/lib.rs` to re-export `SplunkHecExporter` and `SplunkConfig`

    CRITICAL: Use `receipt.timestamp as f64` for the HEC `time` field (Splunk expects epoch seconds as float). The batch format is newline-separated JSON objects, NOT a JSON array -- a JSON array causes HEC to return 400 "Invalid data format".

    2. Define `ElasticAuthConfig` enum:
       - `ApiKey(String)` -- Authorization: ApiKey <key>
       - `Basic { username: String, password: String }` -- Authorization: Basic <base64(user:pass)>

    3. Define `ElasticConfig`:
       - `endpoint: String` -- Elasticsearch endpoint URL (e.g. "https://es.example.com:9200")
       - `index_name: String` -- target index name (default "pact-receipts")
       - `auth: ElasticAuthConfig` -- authentication configuration

    4. Define `ElasticsearchExporter`:
       - Fields: `config: ElasticConfig`, `client: reqwest::Client`
       - `ElasticsearchExporter::new(config: ElasticConfig) -> Result<Self, ExportError>`:
         Build reqwest::Client, return Self.

    5. Implement `Exporter for ElasticsearchExporter`:
       - `name()` returns `"elasticsearch-bulk"`
       - `export_batch(&self, events: &[SiemEvent]) -> Result<usize, ExportError>`:
         a. If events is empty, return Ok(0)
         b. Build NDJSON body: for each event, two lines:
            Line 1: `{"index": {"_index": "<index_name>", "_id": "<receipt.id>"}}`
            Line 2: full receipt JSON (serde_json::to_string(&event.receipt))
            Each line terminated with '\n' (including the last one -- ES bulk requires trailing newline)
         c. POST to `{endpoint}/_bulk`:
            - Header: `Content-Type: application/x-ndjson`
            - Auth header based on ElasticAuthConfig:
              ApiKey: `Authorization: ApiKey {key}`
              Basic: `Authorization: Basic {base64(username:password)}`
            - Body: NDJSON payload
         d. Check HTTP response status: if not 2xx, return ExportError::HttpError
         e. Parse response body as JSON. Check `response["errors"]` field:
            - If `errors == true`: iterate `response["items"]`, count items where `item["index"]["status"]` >= 400
            - If failed_count > 0: return ExportError::PartialFailure { succeeded: total - failed, failed: failed_count, details: first error message }
         f. On full success, return Ok(events.len())

    6. Update `crates/pact-siem/src/exporters/mod.rs` to add `pub mod elastic;`
    7. Update `crates/pact-siem/src/lib.rs` to re-export `ElasticsearchExporter`, `ElasticConfig`, `ElasticAuthConfig`

    CRITICAL: Use receipt.id as _id in the bulk action line. This makes retries idempotent (index action is an upsert). The bulk response returns HTTP 200 even on partial failure -- always parse the response body and check the "errors" field. Use base64 crate (already in pact-cli deps) for Basic auth encoding -- add to pact-siem Cargo.toml if needed or use reqwest's built-in basic_auth method.
