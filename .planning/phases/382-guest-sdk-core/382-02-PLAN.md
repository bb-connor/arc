  <action>
Create `crates/arc-guard-sdk/src/glue.rs` with:

1. Thread-local storage for the last deny reason:
   ```rust
   use std::cell::RefCell;
   
   thread_local! {
       static LAST_DENY_REASON: RefCell<Option<String>> = RefCell::new(None);
   }
   ```

2. `pub unsafe fn read_request(ptr: i32, len: i32) -> Result<GuardRequest, String>`:
   - Creates a slice from `core::slice::from_raw_parts(ptr as *const u8, len as usize)`
   - Calls `serde_json::from_slice(slice).map_err(|e| e.to_string())`
   - Safety doc: caller must ensure ptr and len describe valid UTF-8 JSON in guest memory written by the host

3. `pub fn encode_verdict(verdict: GuardVerdict) -> i32`:
   - Match on verdict:
     - `GuardVerdict::Allow` -> clear LAST_DENY_REASON to None, return VERDICT_ALLOW
     - `GuardVerdict::Deny { reason }` -> store reason in LAST_DENY_REASON, return VERDICT_DENY

4. `#[no_mangle] pub extern "C" fn arc_deny_reason(buf_ptr: i32, buf_len: i32) -> i32`:
   - Read LAST_DENY_REASON from thread-local. If None, return -1.
   - Create `GuestDenyResponse { reason }` and serialize to JSON bytes via `serde_json::to_vec`
   - If serialization fails, return -1
   - If JSON bytes length > buf_len as usize, return -1 (buffer too small)
   - Write JSON bytes into the buffer: `core::slice::from_raw_parts_mut(buf_ptr as *mut u8, buf_len as usize)[..json_bytes.len()].copy_from_slice(&json_bytes)`
   - Return `json_bytes.len() as i32`

   Host contract (from runtime.rs read_structured_deny_reason):
   - buf_ptr = 65536, buf_len = 4096 (fixed constants in host)
   - Return > 0 and <= buf_len: success
   - Return <= 0: no reason available

5. Internal helper `pub(crate) fn clear_deny_reason()` for testing.

6. Unit tests:
   - Test encode_verdict with Allow returns 0, clears reason
   - Test encode_verdict with Deny returns 1, stores reason
   - Test arc_deny_reason after Deny: allocate a buffer, call arc_deny_reason, verify JSON
   - Test arc_deny_reason after Allow returns -1
   - Test arc_deny_reason with tiny buffer returns -1
   - For arc_deny_reason buffer tests on native: allocate a real Vec, pass its pointer as i32 (this is safe on 64-bit only for testing with small allocations in low address space -- alternatively, test via encode_verdict + LAST_DENY_REASON inspection)

   Note: The `read_request` function involves unsafe raw pointer access that is only meaningful on wasm32. Test it indirectly by verifying that a JSON byte slice deserializes correctly using serde_json::from_slice directly, since the unsafe pointer cast is trivial.
  </action>
