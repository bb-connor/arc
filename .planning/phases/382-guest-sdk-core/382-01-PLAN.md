<objective>
Create the arc-guard-sdk crate with guest-side ABI types (GuardRequest,
GuardVerdict, GuestDenyResponse) and a Vec-based guest allocator (arc_alloc,
arc_free). These are the foundational building blocks that all other SDK
modules depend on.

Purpose: Guard authors need typed Rust structs that deserialize identically
to the host's JSON schema, and an allocator the host runtime can call to
place request data in guest linear memory.

Output: New `crates/arc-guard-sdk/` crate with types.rs, alloc.rs, lib.rs,
and Cargo.toml registered in the workspace.

  <action>
1. Create `crates/arc-guard-sdk/src/alloc.rs` with:

   Thread-local storage for keeping Vec allocations alive:
   ```rust
   use std::cell::RefCell;
   
   thread_local! {
       static ALLOCATIONS: RefCell<Vec<Vec<u8>>> = RefCell::new(Vec::new());
   }
   ```

   `arc_alloc` function:
   - Signature: `#[no_mangle] pub extern "C" fn arc_alloc(size: i32) -> i32`
   - If `size <= 0`, return 0
   - Allocate `vec![0u8; size as usize]`, get the raw pointer as i32
   - Push the Vec into ALLOCATIONS thread-local to keep it alive
   - Return the pointer
   - No `unwrap()` or `expect()` anywhere -- return 0 on any failure

   `arc_free` function:
   - Signature: `#[no_mangle] pub extern "C" fn arc_free(ptr: i32, size: i32)`
   - Remove the matching Vec from ALLOCATIONS (match on ptr and len)
   - If no match found, silently ignore (defensive)

   `reset_allocations` internal function (for testing):
   - Clear all allocations from thread-local storage
   - Gate behind `#[cfg(test)]` or `pub(crate)`

   Unit tests in `#[cfg(test)]` module:
   - Test arc_alloc(100) returns non-zero
   - Test arc_alloc(0) returns 0
   - Test arc_alloc(-1) returns 0
   - Test two arc_alloc calls return different pointers
   - Test arc_free does not panic on valid or invalid inputs

   Note on testing: These tests run on the native target (not wasm32).
   The pointer-as-i32 cast is only correct on wasm32 (32-bit pointers),
   but the allocator logic (Vec creation, thread-local storage, free)
   is testable on native. The `#[no_mangle] pub extern "C"` attributes
   work on both targets.
