# Plan 185-02

Define the controlled operating model for supervised-live MERCURY use on the
same workflow.

## Scope

- document the required human roles and ownership boundaries for supervised-live
  operation
- define the fail posture for outages, degraded mode, and escalation while
  customer execution systems remain primary
- align the operating model with Mercury's evidence-first boundary instead of
  implying mediated live control

## Success Criteria

- the supervised-live operating envelope identifies named human roles and
  responsibilities
- outage and degraded-mode handling are explicit and bounded
- the docs state what MERCURY does not take over in controlled production
