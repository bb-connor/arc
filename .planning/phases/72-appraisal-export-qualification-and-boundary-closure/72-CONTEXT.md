# Phase 72: Appraisal Export, Qualification, and Boundary Closure - Context

## Goal

Close `v2.15` with a signed appraisal export surface, multi-adapter
qualification evidence, and honest documentation of ARC's multi-cloud
attestation boundary.

## Why This Phase Exists

`v2.15` only counts if operators and partners can inspect what ARC appraised,
why it trusted or rejected a verifier result, and how the supported
multi-cloud boundary differs from generic attestation interoperability.

## Scope

- signed appraisal export or report artifact
- qualification across Azure, AWS, and Google adapter families
- replay, freshness, rotation, and debug-boundary regression coverage
- milestone audit and planning-state advancement into `v2.16`

## Out of Scope

- adding new verifier families beyond the milestone set
- broad public standards claims beyond the supported profile
- enterprise IAM profile work from the next milestone
