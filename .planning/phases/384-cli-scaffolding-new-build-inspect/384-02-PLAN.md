The function should:

1. Read the .wasm file bytes using `std::fs::read(path)`. If the file does not exist or cannot be read, return `CliError::Other(format!("failed to read {}: {}", path.display(), e))`.

2. Parse the WASM binary using `wasmparser::Parser::new(0)` and iterate over payloads:

```rust
use wasmparser::{Parser, Payload};

let parser = Parser::new(0);
let mut exports: Vec<(String, String)> = Vec::new(); // (name, kind)
let mut memories: Vec<String> = Vec::new();

for payload in parser.parse_all(&wasm_bytes) {
    let payload = payload.map_err(|e| CliError::Other(format!("wasm parse error: {e}")))?;
    match payload {
        Payload::ExportSection(reader) => {
            for export in reader {
                let export = export.map_err(|e| CliError::Other(format!("wasm export parse error: {e}")))?;
                let kind_str = match export.kind {
                    wasmparser::ExternalKind::Func => "function",
                    wasmparser::ExternalKind::Memory => "memory",
                    wasmparser::ExternalKind::Table => "table",
                    wasmparser::ExternalKind::Global => "global",
                    wasmparser::ExternalKind::Tag => "tag",
                };
                exports.push((export.name.to_string(), kind_str.to_string()));
                if export.kind == wasmparser::ExternalKind::Memory {
                    memories.push(export.name.to_string());
                }
            }
        }
        Payload::MemorySection(reader) => {
            for memory in reader {
                let memory = memory.map_err(|e| CliError::Other(format!("wasm memory parse error: {e}")))?;
                let initial_pages = memory.initial;
                let max_pages = memory.maximum.map(|m| format!("{m}")).unwrap_or_else(|| "unbounded".to_string());
                // Store memory info for display (64 KiB per page)
                let initial_kib = initial_pages * 64;
                let info = format!("initial={initial_pages} pages ({initial_kib} KiB), max={max_pages} pages");
                memories.push(info);
            }
        }
        _ => {}
    }
}
```

Note: The above is a sketch. The actual wasmparser 0.221 API uses iterating payloads. Adapt to the exact API -- `Parser::new(0)` takes the offset, `parse_all` returns an iterator of `Result<Payload>`. The `ExportSection` variant wraps a `SectionLimited<Export>` which you iterate. `MemorySection` wraps `SectionLimited<MemoryType>`.

3. After parsing, separate the memory info from export info. Build separate data structures:
   - `export_list`: Vec of (name, kind_str) for all exports
   - `memory_info`: Vec of formatted memory descriptions from MemorySection

4. Check ABI compatibility by looking for required exports:
   - `evaluate` (function) -- main guard entry point
   - `arc_alloc` (function) -- guest memory allocator
   - `arc_deny_reason` (function) -- deny reason retrieval

Mark each as present/missing. Overall ABI is "compatible" if all three are present.

5. Print structured output:

```
=== WASM Guard Inspection ===

File: {path}
Size: {formatted_size}

Exported functions:
  evaluate        (function)
  arc_alloc       (function)
  arc_deny_reason (function)
  arc_free        (function)
  memory          (memory)

ABI compatibility: COMPATIBLE
  [+] evaluate
  [+] arc_alloc
  [+] arc_deny_reason

Memory:
  initial=17 pages (1088 KiB), max=unbounded pages
```

If ABI is not compatible:
```
ABI compatibility: INCOMPATIBLE
  [+] evaluate
  [-] arc_alloc     (MISSING)
  [-] arc_deny_reason (MISSING)
```

6. Return `Ok(())` after printing. The command is informational only -- it does not fail on ABI incompatibility, just reports it.

7. Do NOT use `.unwrap()` or `.expect()`. All wasmparser iteration errors must be handled with `map_err`.
