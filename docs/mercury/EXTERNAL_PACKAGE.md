# MERCURY External Package

**Date:** 2026-04-02  
**Audience:** founders, GTM, design-partner outreach, and fundraising

---

## 1. Canonical External Positioning

Use one clean stack in every short-form external artifact:

- **ARC:** the rights-and-receipts substrate for consequential agent actions
- **MERCURY:** the finance-specific evidence platform built on ARC
- **First workflow:** controlled release, rollback, and inquiry evidence for
  AI-assisted execution workflow changes

Short form:

> MERCURY gives firms signed, portable, review-grade evidence for governed AI
> trading workflows, starting with release, rollback, and inquiry handling.

Do not lead with:

- generic "AI governance"
- full autonomous trading
- best-execution proof
- turnkey regulatory compliance
- broad OMS or EMS replacement

---

## 2. External Artifact Set

The immediate external package should contain four short artifacts.

### A. Company Narrative One-Pager

**Title**

- Review-grade evidence for governed AI trading workflows

**Sections**

1. Problem
   Firms have AI influence in trading workflows, but weak decision evidence
   when a release, rollback, exception, or inquiry needs review.
2. Product
   MERCURY produces signed workflow records, retained artifact references, and
   portable proof and inquiry packages.
3. Why ARC matters
   ARC already supplies the hard substrate: signed receipts, checkpoints,
   exportable evidence, verification, and trust controls.
4. First wedge
   Controlled release, rollback, and inquiry evidence for AI-assisted
   execution workflow changes.
5. Buyer
   Platform engineering, workflow owners, controlled rollout owners, and
   supervisory or review stakeholders.
6. Ask
   Paid design-partner pilot for one controlled workflow.

### B. Design-Partner Brief

**Title**

- Design-partner pilot for governed AI workflow evidence

**Sections**

1. Workflow in scope
   One release, rollback, exception, approval, or inquiry path.
2. Deployment mode
   Replay, shadow, synthetic, or paper-trading first.
3. What MERCURY produces
   Signed receipts, retained artifact references, `Proof Package v1`,
   `Inquiry Package v1`, and verifier output.
4. What the customer supplies
   Workflow definition, source artifacts, approval steps, retention rules,
   reviewer stakeholders, and success metric.
5. Success criteria
   Faster reconstruction, cleaner approval history, safer reviewed exports,
   and a clear productionization decision.
6. Commercial structure
   45-day paid pilot with explicit next-step decision.

### C. Deck Outline

Use a 10-slide deck.

1. ARC + MERCURY
   ARC is the substrate; MERCURY is the finance wedge.
2. The problem
   AI influence is real; decision evidence is weak.
3. The first workflow
   Release, rollback, and inquiry evidence for AI-assisted execution changes.
4. How MERCURY works
   Workflow event -> receipt -> checkpoint -> proof package -> inquiry package.
5. Why this is hard to copy
   ARC substrate, portable proof, chronology, reconciliation, and review-safe
   exports.
6. Product surface
   Signed records, retained artifacts, verifier, and reviewed exports.
7. Buyer and pricing
   Design partner, annual land, and expansion path.
8. Pilot plan
   45 days, one workflow, one measurable operational delta.
9. Expansion
   Same workflow into supervised-live, then governance, downstream consumers,
   and assurance.
10. Ask
   Pilot, partnership, or financing.

### D. Demo Storyboard

The demo should tell one story only.

1. A workflow owner proposes a model, prompt, policy, or parameter change.
2. The system captures source artifacts and approval context.
3. MERCURY emits a signed receipt for proposal or approval state.
4. The workflow is released or rolled back in a controlled path.
5. A checkpoint is published and the record becomes externally verifiable.
6. A reviewer retrieves `Proof Package v1` and verifies it.
7. A reviewer generates `Inquiry Package v1` for a specific audience.
8. The operator shows the exact approved export and associated approval state.

The demo should end with a decision:

- productionize the same workflow
- keep in controlled review mode
- stop

## 3. Messaging Guardrails

Use repeatedly:

- governed AI trading workflows
- review-grade evidence
- signed, portable workflow records
- release, rollback, and inquiry handling
- built on ARC

Avoid repeatedly:

- compliance layer for autonomous trading
- AI Act compliance
- best-execution proof
- universal AI control plane
- generic LLM logging

---

## 4. Asset Production Order

Build the short-form package in this order:

1. company narrative one-pager
2. design-partner brief
3. demo storyboard
4. 10-slide deck

Do not build multiple outward narratives in parallel. Freeze the first workflow
sentence first, then reuse it across every artifact.
